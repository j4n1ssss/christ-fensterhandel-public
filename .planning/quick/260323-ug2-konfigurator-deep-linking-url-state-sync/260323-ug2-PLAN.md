Check nuqs docs: `parseAsArrayOf` might work for comma-separated strings. For the custom oeffnungsarten parser, use `createParser<WingOpening[]>` from nuqs with `serialize` and `parse` methods.

**4. Rewrite `useKonfiguratorUrlState` hook with TWO-PHASE sync:**

The hook must handle the CMS data timing problem. URL params contain slugs, but the store uses IDs. Slug resolution requires CMS data.

**Phase A: Store -> URL (always active, runs on every store change):**

Subscribe to ALL store selection fields. When any changes, convert IDs to slugs using `idToSlug` and update URL via `setUrlState`. Only do this when `cmsData` is loaded (non-null). This is the "write" direction.

Use a single `useEffect` with dependencies on all relevant store fields. Build the URL state object by:
- For each SLUG_FIELD: `idToSlug(cmsData, field.collection, store[field.storeKey])` -> set URL param or null
- For SLUG_ARRAY_FIELDS: map each ID to slug -> join with comma or set null if empty
- For oeffnungsarten: serialize WingOpening[] to custom format (resolve each oeffnungsart ID to slug)
- For masse: `store.masse?.breite` and `store.masse?.hoehe` directly (these are numbers, no slug resolution)
- For gleichWieAussen: direct boolean
- For step: `store.currentStep` directly

IMPORTANT: Use `{ shallow: false }` or equivalent nuqs option to batch URL updates. Use `setUrlState` with the complete object to avoid multiple history entries.

IMPORTANT: To prevent infinite loops, track a `isUrlSeeding` ref. When seeding store from URL (Phase B), set `isUrlSeeding.current = true` and skip the Store->URL effect. Reset after seeding completes.

**Phase B: URL -> Store (runs once after CMS data is available):**

This is the initial seeding from a shared/bookmarked URL. It should run ONCE when:
1. Component mounts
2. CMS data has loaded (`cmsData !== null`)
3. URL has params beyond just `step`

Use a `useEffect` that depends on `cmsData`. When cmsData becomes non-null AND URL has selection params:
- Set `isUrlSeeding.current = true`
- Read all URL params from `urlState`
- For each SLUG_FIELD with a non-null URL value: `slugToId(cmsData, field.collection, urlSlug)` -> `store.setSelection(field.storeKey, resolvedId)`
- For SLUG_ARRAY_FIELDS: split comma-separated slugs, resolve each to ID -> `store.setSelection(field.storeKey, resolvedIds)`
- For oeffnungsarten: parse custom format, resolve slugs to IDs -> `store.setSelection('oeffnungsarten', resolvedWingOpenings)`
- For masse: if both breite and hoehe present, `store.setSelection('masse', { breite, hoehe })`
- For gleichWieAussen: `store.setSelection('gleichWieAussen', value)`
- For step: `store.setStep(urlState.step)`
- After all selections set, set `isUrlSeeding.current = false`

Use a ref `hasSeededFromUrl` to ensure this only runs once per mount, not on every cmsData reference change.

**5. Export the hook and parsers as before:**

Keep the same export signature: `export function useKonfiguratorUrlState()` and `export const konfiguratorParsers`.

**Edge cases to handle:**
- URL has a slug that does not exist in CMS data (e.g., deleted product) -> skip that field silently, do not crash
- URL has partial selections (e.g., only produkttyp and step) -> seed only what is present
- Store changes while URL seeding is in progress -> the `isUrlSeeding` ref prevents Store->URL from firing
