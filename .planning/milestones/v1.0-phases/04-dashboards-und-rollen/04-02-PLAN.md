    **1. Dashboard Overview (src/components/admin/dashboard-overview.tsx):**
    React Server Component (async function). Uses Payload Local API (`getPayload({ config })`) to query Anfragen stats:
    - Stat cards: Neue Anfragen heute (status='neu', createdAt >= today), Offene gesamt (status in ['neu','in_bearbeitung','rueckfrage']), Bestaetigte diesen Monat (status='bestaetigt', createdAt >= month start), Umsatz (sum of gesamtpreis for status='bezahlt' or 'abgeschlossen')
    - Status-Verteilung: Count per status, display as colored badges (use Tailwind: neu=blue, in_bearbeitung=yellow, bestaetigt=green, bezahlt=emerald, abgeschlossen=gray, rueckfrage=orange, abgelehnt=red)
    - Letzte 10 Anfragen: Compact table/list with anfrage_nummer, nachname, status badge, gesamtpreis, createdAt. Each row links to `/admin/collections/anfragen/{id}`
    - Use standard HTML/CSS with Tailwind classes. Payload Admin allows Tailwind in custom views.
    - Import Lucide icons for stat cards (FileText, Clock, CheckCircle, Euro)

    **2. Anfrage Detail View (src/components/admin/anfrage-detail-view.tsx):**
    This is more complex. The user wants a custom single-page view replacing the default edit view for Anfragen.
    - Use 'use client' directive since it needs interactive status buttons
    - Import `useDocumentInfo` from `@payloadcms/ui` to get current document data
    - 3-column layout (responsive: stack on mobile):
      - Left: Status-Timeline component (chronological StatusHistorie entries)
      - Center: Produktliste with Mini-SVG (simplified WindowSVG or just product summary cards with key specs)
      - Right: Kontaktdaten card + Interne Notizen (textarea, auto-saves) + Gesamtpreis
    - Top: Schnell-Status-Buttons via StatusWorkflow component
    - Include a "Zurueck zur Standard-Ansicht" link that opens the default Payload edit view

    **3. Status Workflow (src/components/admin/status-workflow.tsx):**
    Client component ('use client'). Shows available next statuses as buttons:
    - Import `getNextStatuses`, `COMMENT_REQUIRED` from `@/lib/status-transitions`
    - Show only valid next statuses as buttons (colored by status)
    - If target status is in COMMENT_REQUIRED, show a textarea modal/inline before submitting
    - On click: PATCH `/api/anfragen/{id}` with `{ status: newStatus, _status_kommentar: comment }`, credentials: 'include'
    - After success: refresh the page (router.refresh() or window.location.reload())

    **4. Status Timeline (src/components/admin/status-timeline.tsx):**
    Server or client component. Fetches StatusHistorie for current Anfrage:
    - Fetch from `/api/status_historie?where[anfrage][equals]={id}&sort=zeitpunkt`
    - Display as vertical timeline: colored dots, von_status -> zu_status, geaendert_von (email), zeitpunkt formatted, kommentar if present
    - Most recent at top

    **5. Register in payload.config.ts:**
    - Add `admin.components.views.dashboard.Component` pointing to `'@/components/admin/dashboard-overview#default'`
    - In Anfragen collection config, add `admin.components.views.edit.default.Component` pointing to `'@/components/admin/anfrage-detail-view#default'`
    - Run `npx payload generate:importmap` after changes

    Note: Payload 3.x uses string import paths with hash for named exports. Follow the existing AnfrageEditButton pattern.
