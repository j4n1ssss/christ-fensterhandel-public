<task type="auto">
  <name>Task 1: N8N Webhook Utility + afterChange Hook + Fehler-Tracking</name>
  <files>src/lib/n8n-webhook.ts, src/collections/business/anfragen.ts, src/payload-globals/webhook-errors.ts, src/payload.config.ts</files>
  <action>
1. Create `src/lib/n8n-webhook.ts`:
   - Export interface `WebhookPayload`:
     ```
     event_type: 'neue_anfrage' | 'status_aenderung' | 'zahlung_eingegangen'
     anfrage_id: string
     anfrage_nummer: string
     status: { neu: string; alt?: string }
     kunde: { name: string; email: string }
     gesamtbetrag: number
     produkt_anzahl: number
     stripe_checkout_url?: string
     ```
   - Export async function `sendN8NWebhook(payload: WebhookPayload): Promise<boolean>`:
     - Read N8N_WEBHOOK_URL and N8N_WEBHOOK_SECRET from process.env
     - If URL not configured, console.warn and return false (non-blocking)
     - fetch POST to URL with headers: Content-Type application/json, x-webhook-secret: secret
     - Body: JSON.stringify(payload)
     - On success (2xx): return true
     - On failure: console.error with status code and response text, call trackWebhookError (see below), return false
     - On network error: console.error, trackWebhookError, return false
   - Export async function `trackWebhookError(error: string, context: string)`:
     - Import getPayload + config lazily (avoid circular deps)
     - Read webhook_errors Global, append to errors array (keep last 50)
     - Update Global with new error entry: { timestamp: ISO string, error, context }
     - Wrap in try/catch — if Global update fails, just console.error (never throw)

2. Create `src/payload-globals/webhook-errors.ts` — Payload Global config:
   - slug: 'webhook_errors'
   - label: 'Webhook Fehler'
   - admin.group: 'System'
   - access: read/update only admin
   - Fields:
     - `errors`: type json, label 'Fehler-Log', admin.readOnly true
     - `last_error_at`: type date, label 'Letzter Fehler', admin.readOnly true
   - Register this Global in payload.config.ts globals array

3. Rewrite `src/collections/business/anfragen.ts` afterChange hook (replace placeholder):
   - Import `sendN8NWebhook` and `WebhookPayload` from '@/lib/n8n-webhook'
   - Import `createCheckoutSession` from '@/lib/stripe'
   - The hook receives `{ doc, previousDoc, operation }` — add operation to destructuring

   Case A — NEW ANFRAGE (operation === 'create'):
   - Build WebhookPayload with event_type 'neue_anfrage'
   - kunde.name = `${doc.kontaktdaten?.vorname} ${doc.kontaktdaten?.nachname}`.trim()
   - kunde.email = doc.kontaktdaten?.email || ''
   - gesamtbetrag = doc.gesamtpreis || 0
   - produkt_anzahl = doc.produkte?.length || 0
   - status = { neu: doc.status || 'neu' }
   - Call sendN8NWebhook (fire-and-forget, do NOT await to avoid blocking) — actually DO await but wrap in try/catch so it never blocks the main flow

   Case B — STATUS CHANGED (previousDoc && previousDoc.status !== doc.status):
   - Build WebhookPayload with event_type 'status_aenderung'
   - status = { neu: doc.status, alt: previousDoc.status }
   - Same kunde/gesamtbetrag/produkt_anzahl extraction

   Special case: if doc.status === 'bestaetigt':
   - Create Stripe Checkout Session via createCheckoutSession(doc.id, doc.anfrage_nummer, doc.gesamtpreis, doc.produkte?.length || 0)
   - Set stripe_checkout_url = session.url on the payload
   - Wrap in try/catch — if Stripe fails, log error, still send webhook without URL

   - Call sendN8NWebhook with the payload
   - All webhook calls wrapped in try/catch — errors logged but NEVER block the main Payload operation

4. Update payload.config.ts to register the webhook_errors Global:
