## Decisions Made
- Used `(payload as any).findGlobal()` pattern for Navigation/Footer globals (same pattern as existing WebhookErrors access) since generated types don't include new globals until regeneration
- Homepage tries loading Puck 'home' page first with graceful fallback to static landing page, allowing business owner to replace it when ready
- Extended `baseConfig` (server-safe) rather than `editorConfig` (client-only) for the puck-config used in frontend rendering
- Puck plugin auto-generates Pages collection with `versions: { drafts: true }` — no manual collection definition needed
- Custom categories prefixed with 'christ-' to avoid name collisions with plugin's built-in categories

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed TypeScript type casting for Puck page data access**
- **Found during:** Task 2 (Catch-all route)
- **Issue:** Payload `find()` returns typed docs but the auto-generated Pages collection isn't in payload-types.ts yet, causing type errors when accessing `puckData` and `title` fields
- **Fix:** Used `as unknown as Record<string, unknown>` pattern for page data access, and `(payload as any)` for findGlobal calls (consistent with existing codebase pattern)
- **Files modified:** src/app/(frontend)/[...slug]/page.tsx, src/app/(frontend)/page.tsx, src/app/(frontend)/layout.tsx
- **Verification:** `npx tsc --noEmit` passes
- **Committed in:** 0daa468 (Task 2 commit)
