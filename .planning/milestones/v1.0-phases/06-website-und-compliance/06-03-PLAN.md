  <action>
Fix two issues in layout.tsx by detecting locale from the URL pathname.

**Step 1 — Update `src/middleware.ts` to set x-pathname header:**
In the middleware function, ALWAYS set an `x-pathname` header on the request before calling `NextResponse.next()`:
```typescript
const requestHeaders = new Headers(request.headers)
requestHeaders.set('x-pathname', request.nextUrl.pathname)
return NextResponse.next({ request: { headers: requestHeaders } })
```
Ensure this is set on ALL code paths (locale redirects excluded since they return a redirect response, not next()). Both locale-prefixed routes AND non-locale routes (konfigurator, warenkorb) must set this header.

**Step 2 — Update `src/app/(frontend)/layout.tsx` to read locale from header:**
```typescript
import { headers } from 'next/headers'

// Inside RootLayout (async RSC):
const headersList = await headers()
const pathname = headersList.get('x-pathname') || '/'
const localeMatch = pathname.match(/^\/(de|en)(\/|$)/)
const locale = localeMatch ? localeMatch[1] : 'de'
```
Change line 60 from `<html lang="de">` to `<html lang={locale}>`.

**Step 3 — Add locale to findGlobal calls:**
Update both global queries to pass locale:
```typescript
const navGlobal = await (payload as any).findGlobal({ slug: 'navigation', locale: locale as any, fallbackLocale: 'de' as any })
const footerGlobal = await (payload as any).findGlobal({ slug: 'footer', locale: locale as any, fallbackLocale: 'de' as any })
```
The `as any` casts follow the established pattern from 06-01 (see STATE.md decision).
