## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (gap-1, p-1) |
| sm | 8px | Compact element spacing (gap-2, p-2, space-y-2) |
| md | 16px | Default element spacing (gap-4, p-4, space-y-4) |
| lg | 24px | Section padding (p-5 = 20px, p-6 = 24px) |
| xl | 32px | Layout gaps (py-8 = 32px) |
| 2xl | 48px | Major section breaks (space-y-12) |

Exceptions:
- Progress bar dots: 12px diameter (completed/upcoming), 16px diameter (active step) -- not on the 8-point scale but required for visual hierarchy in the stepper
- Mini progress bar dots: 8px diameter -- compact variant for list cards
- Touch targets: minimum 44px for interactive elements on mobile (Stripe button, back link)

## Typography

Two weights only: 400 (normal) and 600 (semibold).

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (normal) | 1.43 (20px) | text-sm |
| Label | 12px | 400 (normal) | 1.33 (16px) | text-xs |
| Heading | 16px | 600 (semibold) | 1.5 (24px) | text-base font-semibold |
| Page Title | 24px | 600 (semibold) | 1.33 (32px) | text-2xl font-semibold |

Source: Consolidated from existing kunden components. Weight 500 (medium) merged into 400 (normal). Weight 700 (bold) merged into 600 (semibold). All UI elements use exactly these 2 weights.

## ProgressStepper Specification

### Full Variant (Detail View)

```
Layout: Horizontal, full width of container (max-w-4xl)
Structure:
  [Dot]---[Line]---[Dot]---[Line]---[Dot]---[Line]---[Dot]---[Line]---[Dot]
  Anfrage        Angebot         Zahlung       Produktion     Lieferung

Dot sizing:
  - Completed: w-3 h-3 (12px), rounded-full, bg-emerald-500
  - Active:    w-4 h-4 (16px), rounded-full, bg-primary, ring-4 ring-primary/20, animate-pulse-slow
  - Upcoming:  w-3 h-3 (12px), rounded-full, bg-gray-300

Line sizing:
  - Height: h-0.5 (2px)
  - Completed segment: bg-emerald-500
  - Upcoming segment: bg-gray-300
  - Flex: flex-1 (lines fill available space between dots)

Labels:
  - Position: below dots, mt-2
  - Font: text-xs (12px)
  - Completed: text-emerald-700 (weight 400, normal)
  - Active: text-foreground font-semibold (weight 600)
  - Upcoming: text-gray-400 (weight 400, normal)
  - Text: "Anfrage", "Angebot", "Zahlung", "Produktion", "Lieferung"

Container:
  - Flex row, items-center
  - Each step is a flex column (items-center) with dot + label
  - Lines are between step columns
  - px-0 (no horizontal padding -- uses parent container)

Responsive:
  - >= sm: horizontal layout with all labels visible
  - < sm: horizontal layout maintained, labels use text-[10px] to fit
```

### Mini Variant (List Cards)

```
Layout: Inline, fits within card row
Structure: 5 dots in a row with 4px gap, no lines, no labels

Dot sizing:
  - Completed: w-2 h-2 (8px), rounded-full, bg-emerald-500
  - Active:    w-2 h-2 (8px), rounded-full, bg-primary, ring-2 ring-primary/20
  - Upcoming:  w-2 h-2 (8px), rounded-full, bg-gray-300

Container:
  - Inline-flex, items-center, gap-1 (4px)
  - Total width: ~56px (5 * 8px dots + 4 * 4px gaps)
  - Replaces StatusBadge in anfragen-liste.tsx (same position)
```

### Props Interface

```typescript
interface ProgressStepperProps {
  /** Current customer phase from STATUS_CUSTOMER_PHASE */
  currentPhase: CustomerPhase | null;
  /** Render mini variant (dots only, no labels) */
  mini?: boolean;
}
```

### Animation

```css
/* Pulse animation for active step -- slower than default Tailwind pulse */
@keyframes pulse-slow {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.7; }
}

.animate-pulse-slow {
  animation: pulse-slow 2s ease-in-out infinite;
}
```

Add to `globals.css` via `@theme` or `@layer utilities` block.

### Terminal Status Behavior

When `currentPhase` is `null` (storniert, abgelehnt):
- ProgressStepper is NOT rendered
- StatusBanner is rendered instead (see below)

## StatusBanner Specification

```
Layout: Full width, rounded-xl, border, padding p-4

Variants:
  - "error" (storniert, abgelehnt): bg-red-50, border-red-200, text-red-800
  - "warning" (rueckfrage, hersteller_problem, zahlungsproblem, reklamation): bg-orange-50, border-orange-200, text-orange-800

Content:
  - Single paragraph: text from STATUS_CUSTOMER_TEXT[status]
  - For rueckfrage: append " Bitte pruefen Sie den Status-Verlauf fuer Details."
  - Font: text-sm font-semibold (weight 600)
  - No CTA buttons (customer self-service is v2)

Position:
  - Detail view: directly below header, before ProgressStepper or instead of it
  - Replaces ProgressStepper when status is terminal (storniert, abgelehnt)
  - Shows ABOVE ProgressStepper when status is non-terminal special (rueckfrage, hersteller_problem, etc.)
```

### Props Interface

```typescript
interface StatusBannerProps {
  /** Raw status key from Anfragen */
  status: string;
}
```

### Rendering Logic

```
IF status is storniert OR abgelehnt:
  Show error banner (red), hide ProgressStepper
ELSE IF status is rueckfrage OR hersteller_problem OR zahlungsproblem OR reklamation:
  Show warning banner (orange), ALSO show ProgressStepper below
ELSE:
  No banner, show ProgressStepper only
```

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Neue Anfrage" (dashboard header) / "Konfigurator starten" (empty state) |
| Empty state heading | "Noch keine Anfragen" |
| Empty state body | "Sie haben noch keine Anfragen erstellt. Starten Sie den Konfigurator!" |

| Back link | "Zurueck zur Uebersicht" |
| Page title (dashboard) | "Meine Anfragen" |
| Page title (detail) | "Anfrage {anfrage_nummer}" |
| Phase labels (stepper) | "Anfrage", "Angebot", "Zahlung", "Produktion", "Lieferung" |

All copy uses real UTF-8 umlauts (ue -> ue is code-only, display shows "ue" etc.). Siezen throughout (Sie/Ihre/Ihnen).

## N8N Integration (Non-Visual)

Phase 21 includes N8N webhook verification but this has no UI component on the kunden side. The contract for N8N:

- Webhook fires on EVERY status change (not just customer-facing)
- WebhookPayload already includes `customer_facing`, `kunden_text`, `kunden_phase` fields (Phase 18)
- N8N workflow filters on `customer_facing: true` to decide email sending
- No new UI elements for N8N -- it is a backend/automation concern
- N8N workflow documentation delivered as markdown (non-UI deliverable)

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Stepper semantics | `role="list"` on container, `role="listitem"` on each step, `aria-current="step"` on active |
| Stepper screen reader | Each dot has `aria-label="{phase} - {erledigt/aktuell/ausstehend}"` |
| Banner semantics | `role="status"` for informational banners, `role="alert"` for error banners (storniert/abgelehnt) |
| Color contrast | Emerald-700 on white: 4.7:1 (AA pass). Red-800 on red-50: 7.2:1 (AAA pass). Orange-800 on orange-50: 6.8:1 (AAA pass) |
| Mini stepper | `aria-label="Fortschritt: {phase}"` on container, dots are decorative (`aria-hidden="true"`) |
