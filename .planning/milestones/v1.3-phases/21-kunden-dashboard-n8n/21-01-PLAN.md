**Create `src/components/kunden/progress-stepper.tsx`** as a client component ("use client" directive) with two exports:

1. **ProgressStepper** component:
   - Props: `{ currentPhase: CustomerPhase | null; mini?: boolean }`
   - Define `const PHASES: CustomerPhase[] = ["Anfrage", "Angebot", "Zahlung", "Produktion", "Lieferung"]`
   - If `currentPhase === null`, return `null` (terminal statuses show StatusBanner instead)
   - Calculate `currentIndex = PHASES.indexOf(currentPhase)`
   - For each phase at index i, determine state:
     - `i < currentIndex` = completed
     - `i === currentIndex` = active
     - `i > currentIndex` = upcoming
   - **Full variant** (mini=false, default):
     - Container: `<div role="list" className="flex w-full items-center">`
     - Each step is a `<div role="listitem">` containing a flex column with dot + label
     - Between steps, render connecting lines: `<div className="h-0.5 flex-1 {bg-emerald-500 if completed, bg-gray-300 if upcoming}" />`
     - Dot sizes: completed = `w-3 h-3 rounded-full bg-emerald-500`, active = `w-4 h-4 rounded-full bg-primary ring-4 ring-primary/20 animate-pulse-slow`, upcoming = `w-3 h-3 rounded-full bg-gray-300`
     - Labels below dots with `mt-2 text-xs`: completed = `text-emerald-700`, active = `text-foreground font-semibold`, upcoming = `text-gray-400`
     - Label text: PHASES[i] ("Anfrage", "Angebot", etc.)
     - Responsive: on < sm use `text-[10px]` for labels
     - Each dot gets `aria-label="{phase} - {erledigt|aktuell|ausstehend}"`, active step gets `aria-current="step"`
   - **Mini variant** (mini=true):
     - Container: `<div className="inline-flex items-center gap-1" aria-label="Fortschritt: {currentPhase}">`
     - 5 dots only, no lines, no labels
     - All dots `w-2 h-2 rounded-full`: completed = `bg-emerald-500`, active = `bg-primary ring-2 ring-primary/20`, upcoming = `bg-gray-300`
     - All dots `aria-hidden="true"` (container has the label)

2. **ProgressStepperMini** convenience export:
   - `export function ProgressStepperMini({ currentPhase }: { currentPhase: CustomerPhase | null }) { return <ProgressStepper currentPhase={currentPhase} mini /> }`

   ```typescript
   const TERMINAL_STATUSES = ["storniert", "abgelehnt"] as const
   const WARNING_STATUSES = ["rueckfrage", "hersteller_problem", "reklamation"] as const
   const ERROR_STATUSES = ["zahlungsproblem"] as const
   ```

3. **StatusBanner** component with props `{ status: string }`:
   - Determine variant:
     - `isTerminal = TERMINAL_STATUSES.includes(status as any)`
     - `isWarning = WARNING_STATUSES.includes(status as any)`
     - `isError = ERROR_STATUSES.includes(status as any) || isTerminal`
   - If none of `isTerminal`, `isWarning`, `isError` is true, return `null` (normal statuses get no banner)
   - Get text: `const text = STATUS_CUSTOMER_TEXT[status as StatusKey] ?? ""`
   - For rueckfrage, append: `" Bitte pruefen Sie den Status-Verlauf fuer Details."`
   - Determine colors:
     - error variant (isError): `bg-red-50 border-red-200 text-red-800`
     - warning variant (isWarning): `bg-orange-50 border-orange-200 text-orange-800`
   - Determine ARIA role:
     - Terminal statuses (storniert, abgelehnt): `role="alert"`
     - All others: `role="status"`
   - Render:
     ```tsx
     <div role={role} className={`rounded-xl border p-4 ${colorClasses}`}>
       <p className="text-sm font-semibold">{displayText}</p>
     </div>
     ```

Note: No CTA buttons in banners (Kunden-Antwort is v2/deferred).
