# Phase 19 — UI Design Contract

> Visual and interaction contract for the Admin Anfrage Detail View Redesign. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Important:** This phase operates entirely within the Payload Admin Panel. Tailwind CSS and Shadcn UI are NOT used. All styling uses inline styles + admin-custom.css with Payload's `var(--theme-*)` CSS variables.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Payload Admin Panel scope) |
| Preset | not applicable |
| Component library | none (vanilla React with inline styles + admin-custom.css) |
| Icon library | none (no icons -- text labels only, per project decision) |
| Font | Payload Admin default (system font stack) |

**Rationale:** Payload Admin Panel does not use Tailwind. All admin components use inline styles with `var(--theme-*)` Payload CSS variables for dark/light theme compatibility. Phase 19 introduces `admin-custom.css` for structural layout classes to replace repetitive inline styles (ADMN-01).

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, border-left urgency indicator, spec-grid row padding |
| sm | 8px | Compact element spacing, inline padding, gap between spec rows, dropdown item padding, product card header gap, attention bar element gap |
| md | 16px | Default element spacing, card padding, tab panel padding, two-column layout gap |
| lg | 24px | Section padding, page padding, attention bar padding |
| xl | 32px | Quantity badge dimensions (32x32px), major component gaps |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Quantity badge: 32x32px (xl token)
- Touch targets: minimum 36px height for all interactive buttons (splitbutton, tab triggers, save buttons)
- Attention bar left border: 4px (xs token) -- urgency color indicator

---

## Typography

All sizes reference the Payload Admin system font stack. No custom fonts. Exactly 4 sizes and 2 weights.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 13px | 400 | 1.5 | Spec grid values, contact details, notes text, dropdown items, fluegel info append |
| Label | 12px | 600 | 1.4 | Section headers uppercase, "Gesamtpreis" caption, tab labels, urgency badge text, anonymisieren button, spec row labels |
| Heading | 14px | 600 | 1.3 | Card section titles ("Kontakt", "Timeline", "Notizen"), product identity line, product dimensions line (Breite x Hoehe), quantity badge text |
| Display | 20px | 600 | 1.2 | Anfrage-Nummer in attention bar, Gesamtpreis in attention bar |

**Weight mapping (2 weights only):**
- 400: All body text, spec values, secondary information
- 600: All headings, labels, badges, emphasized content, display text, price totals

## Component Inventory

### 1. Attention Bar (`attention-bar`)

Full-width bar at the top of the detail view. Displays at-a-glance overview.

**Layout:** Flexbox, `justify-content: space-between`, `align-items: center`, `flex-wrap: wrap`, `gap: 8px`
**Background:** `var(--theme-elevation-50)`
**Border:** `1px solid var(--theme-elevation-200)`
**Border-left:** `4px solid {urgency_color}` (only for warn/urgent/critical)
**Border-radius:** `8px`
**Padding:** `16px 24px`

**Content slots (left to right):**

| Slot | Content | Style |
|------|---------|-------|
| Anfrage-Nummer | `#2024-047` | 20px, weight 600, `var(--theme-elevation-1000)` |
| Status-Badge | Farbiger Pill mit STATUS_LABELS Text | 13px, weight 600, `${statusColor}18` bg, `statusColor` text, `${statusColor}40` border, border-radius 16px, padding 4px 16px |
| Wartezeit-Badge | "Wartet 3 Tage" / "DRINGEND -- 8 Tage" | 12px, weight 600, urgency color text, urgency bg at 12% opacity, padding 4px 8px, border-radius 4px |
| Produkt-Zusammenfassung | "2x Fenster, 1x Balkontuer" | 13px, weight 400, `var(--theme-elevation-600)` |
| Gesamtpreis | "2.480,00 EUR" | 20px, weight 600, `var(--theme-elevation-1000)` |

**Wartezeit computation:**
```
days = Math.floor((Date.now() - new Date(last_status_change_at).getTime()) / 86_400_000)
```
- Uses `last_status_change_at` field (Phase 18), NOT `createdAt`
- Terminal statuses (storniert, abgeschlossen): No urgency badge, show completion date instead

**Produkt-Zusammenfassung computation:**
```
Group produkte[] by produkttyp, sum stueckzahl per group.
Format: "{count}x {produkttyp}" joined with ", "
Example: "2x Fenster, 1x Balkontuer"
```

## Interaction Contracts

### Status Change Flow

1. User clicks splitbutton primary -> if NOT COMMENT_REQUIRED: immediate PATCH to `/api/anfragen/{id}` with `{ status: targetStatus }` -> on success: `refreshKey++` to reload doc + timeline
2. User clicks splitbutton primary -> if COMMENT_REQUIRED: comment panel expands inline below splitbutton -> user types comment -> clicks "Status aendern" -> PATCH with `{ status: targetStatus, _status_kommentar: comment }` -> on success: collapse panel, `refreshKey++`
3. User clicks chevron -> dropdown opens -> clicks secondary action -> same COMMENT_REQUIRED check as step 1/2
4. User clicks "Anfrage stornieren" -> `window.confirm()` fires -> if confirmed: stornierung fields panel expands -> user fills stornierung_grund (required) + rueckerstattung fields (if paid) -> PATCH with all fields -> on success: refresh
5. Dropdown closes on: outside click, Escape key, successful action

### Tab Switching

1. User clicks tab trigger -> active tab updates immediately (no animation)
2. Active tab written to `sessionStorage` key `anfrage-detail-tab`
3. On page load: read from sessionStorage, validate tab exists, set active or fallback to "Kontakt"
4. Details tab: hidden from tab bar when status has no relevant fields

### Data Loading

1. Component mounts -> fetch `/api/anfragen/{id}?depth=1` with credentials
2. On success: set doc, set notizen from doc.interne_notizen
3. On status change: increment refreshKey -> useEffect re-fetches
4. Loading state: "Lade Anfrage..." with `var(--theme-elevation-500)` text
5. Not found state: "Anfrage nicht gefunden."

### Notizen Save

1. User edits textarea -> local state updates
2. User clicks "Notizen speichern" -> PATCH `/api/anfragen/{id}` with `{ interne_notizen: notizen }`
3. Button shows "Wird gespeichert..." with opacity 0.6 during save
4. No success toast -- button returns to normal state

### Anonymization

1. User clicks "Kundendaten anonymisieren (DSGVO)" -> window.confirm with warning text
2. If confirmed: POST `/api/admin/anonymize-customer` with `{ anfrageId: id }` -> on success: refreshKey++
3. If error: `alert()` with error message
4. Button hidden when contact already anonymized (vorname === "GELOESCHT")
