    const [dropdownOpen, setDropdownOpen] = useState(false);

    const [error, setError] = useState<string | null>(null);

    Imports:
    - `React, { useState, useRef, useEffect }` from "react"
    - `QUICK_ACTIONS, STATUS_COLORS, STATUS_LABELS` from `@/lib/status-config`
    - `COMMENT_REQUIRED` from `@/lib/status-transitions`
    - `isTerminalStatus, isCompletedStatus` from `@/lib/detail-view-helpers`

    Core logic:

    1. Get actions: `const actions = QUICK_ACTIONS[currentStatus as StatusKey] || []`
    2. If `isTerminalStatus(currentStatus)`: render terminal info text "Storniert am {date}" using `<div className="terminal-info">`. No splitbutton.
    3. If `isCompletedStatus(currentStatus)`: render "Diese Anfrage ist abgeschlossen." + "Wieder öffnen" link as `<button className="terminal-info__reopen">` that calls `handleActionClick("wieder_geoeffnet")`. No splitbutton.
    4. If `actions.length === 0`: return null
    5. `primaryAction = actions[0]`
    6. `secondaryActions = actions.slice(1)`
    7. `primaryColor = STATUS_COLORS[primaryAction.target as StatusKey] || "#6b7280"`
    8. Darker color for chevron: darken by adjusting brightness -- simplest approach: just use same color with added border-left separator

    Outside-click handler (useEffect):
    ```typescript
    useEffect(() => {
      if (!dropdownOpen) return;
      const handleClick = (e: MouseEvent) => {
        if (dropdownRef.current && !dropdownRef.current.contains(e.target as Node)) {
          setDropdownOpen(false);
        }
      };
      const handleEscape = (e: KeyboardEvent) => {
        if (e.key === "Escape") setDropdownOpen(false);
      };
      document.addEventListener("mousedown", handleClick);
      document.addEventListener("keydown", handleEscape);
      return () => {
        document.removeEventListener("mousedown", handleClick);
        document.removeEventListener("keydown", handleEscape);
      };
    }, [dropdownOpen]);
    ```

    `handleActionClick(targetStatus: string)`:
    - Close dropdown: `setDropdownOpen(false)`
    - Reset error: `setError(null)`
    - **Stornierung special rule**: if `targetStatus === "storniert"`:
      - Call `window.confirm("Stornierung ist endgültig. Fortfahren?")`
      - If NOT confirmed: return (do nothing)
      - If confirmed: `setShowCommentFor("storniert")` to show stornierung fields panel
      - Return (don't proceed to normal flow)
    - If `COMMENT_REQUIRED.includes(targetStatus)`:
      - `setShowCommentFor(targetStatus)` and `setComment("")`
    - Else:
      - Call `submitStatusChange(targetStatus, "")`

    `submitStatusChange(targetStatus: string, statusComment: string)`:
    - Reuse the exact logic from status-workflow.tsx lines 35-73:
      - `setSaving(true); setError(null)`
      - PATCH `/api/anfragen/${anfrageId}` with body:
        - For storniert: `{ status: "storniert", stornierung_grund: stornierungGrund, rueckerstattung_betrag: rueckerstattungBetrag ? Number(rueckerstattungBetrag) : undefined, rueckerstattung_status: rueckerstattungStatus || undefined }`
        - For others: `{ status: targetStatus, _status_kommentar: statusComment || undefined }`
      - On success: reset all state, call `onStatusChanged()`
      - On error: extract message from response, set `setError(errMsg)`, with fallback "Fehler beim Statuswechsel -- bitte Seite neu laden und erneut versuchen." (use real em-dash)
      - On catch: `setError("Netzwerkfehler -- bitte Internetverbindung prüfen und erneut versuchen.")`
      - Finally: `setSaving(false)`
