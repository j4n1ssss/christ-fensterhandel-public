# Phase 11 -- UI Design Contract

> Visual and interaction contract for Payload Admin components: ProfileLastEditor header line and ProfileHistoryPanel document view tab. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Payload CMS Admin UI (inline styles with Payload CSS variables) |
| Preset | Payload default theme (`--theme-elevation-*` scale) |
| Component library | `@payloadcms/ui` hooks (useDocumentInfo, useForm) |
| Icon library | lucide-react (already installed, used by ProfileEditToolbar) |
| Font | Payload Admin default (system font stack -- no override) |

**Context:** This phase creates components that render INSIDE the Payload Admin panel, not on the customer-facing frontend. All existing admin components (ProfileEditToolbar, StatusTimeline, AnfrageDetailView) use inline styles with Payload CSS custom properties. This contract follows the same pattern.

**Primary focal point:** The event badge (colored, left-aligned) on each entry row draws the eye first, orienting the viewer to the type of change before they read timestamp and editor details.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, inline padding, tooltip margin |
| sm | 8px | Compact element spacing, chevron padding, tag gaps |
| md | 16px | Default panel padding, section spacing |
| lg | 24px | Timeline left padding (matches StatusTimeline pattern) |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch targets for expand/collapse chevron buttons: 28px minimum click area (not a standalone touch target, embedded in row).

## Typography

| Role | Size | Weight | Line Height | Payload Variable |
|------|------|--------|-------------|------------------|
| Body | 13px | 400 (regular) | 1.5 | Matches StatusTimeline body text |
| Label / Meta | 11px | 400 (regular) | 1.4 | Timestamp, editor name, field label badges |
| Heading | 13px | 600 (semibold) | 1.4 | Entry header (event type), section headings |
| Field Badge | 11px | 600 (semibold) | 1.0 | Changed field name badges |

**Note:** Payload Admin uses 13px as its standard body size. This phase follows that convention exactly, matching StatusTimeline (13px body, 11px meta, 12px quotes). No 14px or 16px sizes -- those would look inconsistent within Payload's UI.

## Color

All colors use Payload's `--theme-elevation-*` CSS custom properties for automatic dark mode support.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--theme-elevation-0)` | Panel backgrounds, entry backgrounds |
| Secondary (30%) | `var(--theme-elevation-100)` | Diff detail backgrounds, expanded content area, field badge backgrounds |
| Accent (10%) | `var(--theme-elevation-900)` | Entry text, event labels, headings |
| Muted text | `var(--theme-elevation-500)` | Timestamps, editor names, empty state text |
| Border | `var(--theme-elevation-200)` | Entry separators, panel borders |
| Hover | `var(--theme-elevation-50)` | Row hover state background |
| Destructive | `#ef4444` | Not used in this phase (no destructive actions) |

### Semantic Colors for Diff Display

| Role | Value | Usage |
|------|-------|-------|
| Added value background | `rgba(34, 197, 94, 0.08)` / `#22c55e08` | "to" value background in diff detail |
| Added value text | `#16a34a` | "to" value text color |
| Added value border | `#22c55e` | Left border on "to" value block |
| Removed value background | `rgba(239, 68, 68, 0.08)` / `#ef444408` | "from" value background in diff detail |
| Removed value text | `#dc2626` | "from" value text color |
| Removed value border | `#ef4444` | Left border on "from" value block |
| Event badge: update | `rgba(59, 130, 246, 0.12)` text `#3b82f6` | "update" event type badge |

Accent (`--theme-elevation-900`) used for high-contrast text, labels, and headings. Colored accent backgrounds (blue/green/gray badges) reserved for event type badges only.

## Component Inventory

### Component 1: ProfileLastEditor (beforeDocumentControls)

**Location:** Renders in the `beforeDocumentControls` array alongside existing ProfileEditToolbar.
**Data source:** Reads `last_edited_by` and `updatedAt` from the document data (no extra API call).

**Layout:**
```
[Undo] [Redo]  |  Zuletzt bearbeitet von Max Mueller (admin@christ.de) am 19.03.2026, 14:30
```

| Property | Value |
|----------|-------|
| Container | `display: flex`, `align-items: center`, `gap: 8px` |
| Separator | `1px solid var(--theme-elevation-200)`, height 16px, `margin: 0 8px` (vertical divider between toolbar and editor info) |
| Text | 11px, weight 400, color `var(--theme-elevation-500)` |
| Name portion | 11px, weight 600, color `var(--theme-elevation-700)` |

**States:**
- **Loaded with editor:** "Zuletzt bearbeitet von **Max Mueller** (admin@christ.de) am 19.03.2026, 14:30"
- **Loaded without name (fallback):** "Zuletzt bearbeitet von admin@christ-fensterhandel.de am 19.03.2026, 14:30"
- **New document (no editor yet):** Component renders nothing (returns `null`)
- **Editor relationship unresolved (ID only):** Component renders nothing (returns `null`) -- this handles the case where depth=0 returns only an ID string

### Component 2: ProfileHistoryPanel (Document View Tab)

**Location:** Custom document view tab at route `/profile/{id}/history`. Tab label: "Historie".
**Data source:** REST API fetch to `/api/edit_history?where[collection][equals]=profile&where[doc_id][equals]={id}&sort=-timestamp&limit=50&depth=1` on mount.

**Layout -- Entry List (collapsed):**
```
+-----------------------------------------------------------+
|  Historie                                                  |
+-----------------------------------------------------------+
|  [update]  19.03.2026, 14:30 -- Max Mueller (admin@...)  |
|  name_einfach  erlaubte_farben  +2 weitere                 |

|  [create]  19.03.2026, 10:15 -- Max Mueller (admin@...)  |
|  Erstellt                                                  |
|                                                     [v]    |
+-----------------------------------------------------------+
```

**Layout -- Entry (expanded):**

#### Entry Row (collapsed) Specification

| Property | Value |
|----------|-------|
| Container | `padding: 12px 16px`, `border-bottom: 1px solid var(--theme-elevation-200)`, `cursor: pointer` |
| Hover | `background: var(--theme-elevation-50)` |
| Event badge | `padding: 4px 8px`, `border-radius: 4px`, `font-size: 11px`, `font-weight: 600`, background and text per event type (see Color table) |
| Timestamp | 11px, weight 400, color `var(--theme-elevation-500)` |
| Editor name | 11px, weight 400, color `var(--theme-elevation-500)` |
| Separator dash | ` -- ` between timestamp and editor |
| Field badges row | `margin-top: 4px`, `display: flex`, `flex-wrap: wrap`, `gap: 4px` |
| Field badge | `padding: 4px 8px`, `border-radius: 4px`, `font-size: 11px`, `font-weight: 600`, `background: var(--theme-elevation-100)`, `color: var(--theme-elevation-700)` |
| Overflow badge | Same style as field badge but text reads "+N weitere" when more than 3 fields changed |
| Chevron icon | `ChevronDown` from lucide-react, size 14px, color `var(--theme-elevation-400)`, positioned `margin-left: auto` |
| Transition | Chevron rotates 180deg when expanded, `transition: transform 0.15s ease` |

#### Entry Detail (expanded) Specification

| Property | Value |
|----------|-------|
| Container | `padding: 8px 16px 16px 16px`, `background: var(--theme-elevation-50)` |
| Field group | `margin-bottom: 12px` (last child `margin-bottom: 0`) |
| Field name | 11px, weight 600, color `var(--theme-elevation-700)`, `margin-bottom: 4px` |
| "From" block | `padding: 4px 8px`, `border-left: 3px solid #ef4444`, `background: rgba(239, 68, 68, 0.08)`, `font-size: 12px`, `color: #dc2626` |
| "To" block | `padding: 4px 8px`, `border-left: 3px solid #22c55e`, `background: rgba(34, 197, 94, 0.08)`, `font-size: 12px`, `color: #16a34a`, `margin-top: 4px` |
| Prefix | "-" for from values, "+" for to values, weight 600, displayed inline before the value text |
| Relationship values | Comma-separated labels: "Weiss RAL 9016, Anthrazit RAL 7016" |
| Empty relationship | Display as "(leer)" in italic |
| Null/undefined simple value | Display as "(leer)" in italic |
| Expand animation | `max-height` transition from 0 to `auto` (or use conditional rendering without animation -- Claude's discretion) |

#### Panel-level States

| State | Rendering |
|-------|-----------|
| Loading | "Lade Historie..." -- 13px, color `var(--theme-elevation-500)`, padding 16px (matches StatusTimeline loading state) |
| Empty (no entries) | "Noch keine Aenderungen protokolliert." -- 13px, color `var(--theme-elevation-500)`, padding 16px |
| Error (fetch failed) | "Historie konnte nicht geladen werden." -- 13px, color `var(--theme-elevation-500)`, padding 16px. No retry button -- user can switch tabs to retry. |
| Loaded with entries | Entry list as specified above |

## Interaction Contract

### ProfileLastEditor
| Interaction | Behavior |
|-------------|----------|
| Click | Not interactive -- display only |
| Data freshness | Reads from form/document data on render. Updates when document is refreshed (after save). |

### ProfileHistoryPanel
| Interaction | Behavior |
|-------------|----------|
| Tab click | Navigates to `/profile/{id}/history` route. Component mounts and fetches data. |
| Tab switch away | Component unmounts. No cached state. |
| Tab switch back | Component re-mounts and re-fetches (fresh data after potential save). |
| Entry click (collapsed) | Expands the entry to show diff details. Chevron rotates 180deg. |
| Entry click (expanded) | Collapses the entry. Chevron rotates back. |
| Multiple entries expanded | Allowed -- accordion behavior is NOT enforced (multiple can be open simultaneously). |
| Row hover | Background changes to `var(--theme-elevation-50)`. |
| Keyboard | No keyboard shortcuts in this phase. Tab navigation follows browser defaults. |
| Scroll | Panel content scrolls naturally within Payload's document view container. No custom scrollbar. |
