# Phase 10 — UI Design Contract: Undo/Redo

> Visual and interaction contract for Phase 10. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Context Note

This phase implements UI exclusively inside the **Payload Admin Panel**, not the customer-facing frontend. Payload's Admin Panel renders with its own CSS variable system (`--theme-elevation-*`, `--theme-bg`). All admin components in this project use inline styles with those variables — not Tailwind classes or shadcn components. This is the established project pattern (see `src/components/admin/status-workflow.tsx`, `anfrage-detail-view.tsx`).

shadcn/radix is initialized and available for the customer-facing frontend (phases 2-6) but is NOT used inside Payload Admin components. The Undo/Redo toolbar must blend visually with Payload's own document controls toolbar.

**Registry safety gate:** No third-party registry blocks are required. All needed APIs are from `@payloadcms/ui` (already installed).

## Toolbar Visual Spec (ProfileEditToolbar)

**Layout:** Two icon-only buttons in a flex row with 4px gap. No visible container/card. Renders inline as `beforeDocumentControls` — the browser places it left of Payload's own Save/Publish buttons.

**Button anatomy (each button):**
- Size: 32px × 32px minimum touch target
- Icon: 16px Lucide icon (`Undo2` for Undo, `Redo2` for Redo)
- Background: transparent at rest
- Border: 1px solid `var(--theme-elevation-200)` at rest
- Border radius: 4px (matching Payload button radius)
- Cursor: pointer when enabled, not-allowed when disabled

**Enabled state:**
- Icon color: `var(--theme-elevation-900)`
- Border: 1px solid `var(--theme-elevation-200)`
- Background: transparent
- Opacity: 1.0

**Disabled state (stack empty or at floor):**
- Icon color: `var(--theme-elevation-300)`
- Border: 1px solid `var(--theme-elevation-150)`
- Background: transparent
- Opacity: 0.4
- Pointer events: none

**Hover state (enabled only):**
- Background: `var(--theme-elevation-50)`
- Border: 1px solid `var(--theme-elevation-300)`
- Transition: all 0.15s (matching project's established transition from `status-workflow.tsx`)

**Tooltip (on hover, enabled state only):**
- Appears after 400ms hover delay
- Content: "Rückgängig (⌘Z)" for Undo, "Wiederherstellen (⌘⇧Z)" for Redo
- Style: 12px, regular weight, dark background `var(--theme-elevation-900)`, white text `var(--theme-elevation-0)`, 4px border-radius, 4px 8px padding
- Position: below the button (tooltip-bottom)

Source: All specs from CONTEXT.md (locked decision: "Buttons im Payload-nativen Stil (ghost/outline) — kein visueller Stilbruch").

## Field Highlight Spec

When Undo or Redo executes and restores form state, changed fields receive a transient visual highlight:

- **Trigger:** Immediately after `dispatchFields({ type: 'REPLACE_STATE', ... })` resolves
- **Mechanism:** Add a CSS class (e.g., `undo-redo-highlight`) to each changed field's DOM wrapper via `document.querySelector` on the field's `data-field-path` attribute
- **Style:** Yellow background flash — `background-color: #eab30833` (yellow-500 at ~20% opacity)
- **Duration:** 500ms
- **Exit animation:** CSS transition `background-color 500ms ease-out` back to transparent
- **Cleanup:** Remove the class after the transition ends (transitionend event or 600ms timeout)
- **Fallback:** If field DOM element is not found, skip highlight silently (not critical path)

Source: CONTEXT.md decision "Feld-Highlight: Geaenderte Felder flashen kurz gelb (Warnung-Farbe, ~500ms)".

## Toast Spec

Single toast notification appears on every Undo or Redo action:

- **API:** `toast.info(message, { duration: 2000 })` from `@payloadcms/ui` (re-exports sonner)
- **Position:** Managed by Payload's existing Toaster — no new Toaster setup needed
- **Duration:** 2000ms (auto-dismiss)
- **Content patterns:**
  - Undo: `"N Feld rückgängig gemacht"` (N=1) / `"N Felder rückgängig gemacht"` (N>1)
  - Redo: `"N Feld wiederhergestellt"` (N=1) / `"N Felder wiederhergestellt"` (N>1)
  - No undo available: no toast (button is disabled — should never reach this state)
- **N = changed field count:** Number of top-level field paths that differ between the restored snapshot and the previous state (computed from diff before dispatch)

Source: CONTEXT.md decision "Toast: Kleine Benachrichtigung mit Aktion + Feldanzahl".

## Interaction States Summary

| State | Undo Button | Redo Button | Keyboard |
|-------|-------------|-------------|----------|
| Stack empty (initial) | Disabled (opacity 0.4) | Disabled (opacity 0.4) | No-op |
| After first edit | Enabled | Disabled (nothing to redo) | Cmd+Z active |
| After Undo | Enabled (if more undos) / Disabled (at floor) | Enabled | Both active |
| After Redo | Enabled | Enabled (if more redos) / Disabled (at top) | Both active |
| At save checkpoint (floor) | Disabled | Depends on current position | Cmd+Z no-op |
| Inside relationship drawer (editDepth > 1) | Disabled via useHotkey | Disabled via useHotkey | Hotkeys suppressed |
