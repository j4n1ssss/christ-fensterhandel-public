```markdown
# ADR-001: Kein versions:drafts in v1.1

**Status:** Accepted
**Datum:** 2026-03-20
**Betrifft:** src/collections/produkte/profile.ts

## Kontext

Payload CMS 3 bietet ein Versions/Drafts-System (`versions: { drafts: true }`), das automatisch Draft/Published-Workflows ermoeglicht. Fuer die Profile-Collection wurde evaluiert, ob dieses Feature in v1.1 aktiviert werden soll.

## Entscheidung

Versions/Drafts wird in v1.1 NICHT aktiviert. Die Profile-Collection bleibt ohne `versions`-Konfiguration.

## Begruendung

Payload CMS fuegt bei Aktivierung von `versions: { drafts: true }` automatisch ein `_status`-Feld (Typ: `select`, Optionen: `draft`/`published`) zu allen Dokumenten hinzu. Bestehende Dokumente haben `_status = null`, da das Feld vor der Aktivierung nicht existierte.

**Das Problem:** Payloads Standard-Query filtert nach `_status = 'published'`. Bestehende Profile mit `_status = null` werden dadurch aus allen Queries ausgeschlossen -- sie sind faktisch unsichtbar. Weder im Admin-Panel noch ueber die API werden sie zurueckgegeben.

**Migration-Risiko:**
- Alle ~20+ bestehenden Profile muessten gleichzeitig auf `_status = 'published'` migriert werden
- Ein Fehler in der Migration macht das gesamte Konfigurator-System unbenutzbar
- Kein Rollback-Mechanismus ohne manuelle Datenbank-Eingriffe
- Betrifft produktive Daten (Admin hat bereits Profile befuellt)

## Konsequenzen

**Positiv:**
- Kein Risiko fuer bestehende Daten
- Kein zusaetzlicher Migrations-Aufwand
- Profile-System bleibt stabil und vorhersagbar

**Negativ:**
- Kein Draft/Published-Workflow fuer Profile in v1.1
- Kein automatisches Version-Tracking durch Payload (wird durch eigenes edit_history-System abgedeckt)

## Alternativen betrachtet

1. **Versions aktivieren mit Pre-Migration** (VRSN-01, deferred to v1.2): _status-Feld vorab migrieren, dann aktivieren. Aufwand: eigener Migrations-Plan, Testing, Rollback-Strategie.
2. **Versions nur fuer neue Profile**: Nicht moeglich -- Payload aktiviert versions collection-weit, nicht pro Dokument.

## Verwandte Dokumente

- `.planning/REQUIREMENTS.md` -- VRSN-01 (deferred to v1.2)
- `.planning/ROADMAP.md` -- Phase 12 Out-of-Scope
- `docs/todos/008_2026-03-15_admin-panel-umbau-plan.md` -- Original-Analyse
```
