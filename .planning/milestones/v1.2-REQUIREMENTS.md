# Requirements: Christ Fensterhandel

**Defined:** 2026-03-23
**Core Value:** Kunden konfigurieren Fenster/Tueren Schritt fuer Schritt mit intelligenter Hub-Filterung -- Christ bekommt strukturierte Anfragen ins Dashboard

## v1.2 Requirements

Requirements fuer Admin-Navigation Umbau. Jedes Requirement mapped auf Roadmap-Phasen.

### Navigation

- [x] **NAV-01**: Admin sieht Custom Sidebar mit Dashboard, Bestellungen, Produkte, Benutzer als direkte Links oben
- [x] **NAV-02**: Admin kann Bestellungsverwaltung aufklappen und sieht Anfragen + Status-Historie
- [x] **NAV-03**: Admin kann Produktverwaltung aufklappen und sieht alle Collection-Links gruppiert nach Hauptprodukte/Ausstattung/Konfiguration/Preise
- [x] **NAV-04**: Admin kann Website aufklappen und sieht Pages, Navigation, Footer, Puck Templates
- [x] **NAV-05**: Admin kann System aufklappen und sieht Medien, Edit-History, Webhook Fehler
- [x] **NAV-06**: Produktverwaltung-Dropdown zeigt graue, nicht-klickbare Untergruppen-Ueberschriften (HAUPTPRODUKTE, AUSSTATTUNG, KONFIGURATION, PREISE)
- [x] **NAV-07**: Navigation enthaelt keine Emojis, nur Text-Labels
- [x] **NAV-08**: Nav-Reihenfolge entspricht exakt dem Mockup

### UX

- [x] **UX-01**: Aktiver Nav-Link ist visuell hervorgehoben (Pill-Style, User-approved 2026-03-23)
- [x] **UX-02**: Dropdown-Auf/Zu-Zustand bleibt waehrend der Session erhalten
- [x] **UX-03**: Nav-Links werden basierend auf User-Rolle gefiltert (visibleEntities)

### Integration

- [x] **INT-01**: WebhookFehlerBadge ist in der Custom Nav sichtbar (afterNavLinks ersetzt)
- [x] **INT-02**: Custom Nav ist in payload.config.ts registriert und importMap generiert
