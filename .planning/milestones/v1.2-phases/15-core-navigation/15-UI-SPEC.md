## Spacing Scale

Declared values (must be multiples of 4). This phase operates INSIDE Payload's admin sidebar which has its own padding variables. Spacing below applies to custom nav content only.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between badge and label text, inline icon margin |
| sm | 8px | Vertical gap between nav items within a section |
| md | 16px | Horizontal padding for nav links (left/right inset), vertical padding between dropdown sections, section header padding |
| lg | 24px | Top/bottom padding of the entire nav scroll area |
| xl | 32px | Reserved (not used in Phase 15) |
| 2xl | 48px | Reserved (not used in Phase 15) |

Exceptions:
- Active link left accent bar width: 3px (not a spacing token, a border width)
- Minimum clickable height for all interactive elements: 36px (not a spacing token, a hardcoded CSS `min-height` value for accessibility touch target compliance per WCAG 2.2)
- Subgroup heading minimum height: 24px (uses spacing token `lg`)
- Payload's own `--nav-padding-inline-start`, `--nav-padding-inline-end`, `--nav-padding-block-start`, `--nav-padding-block-end` control the outer wrapper — do NOT override these

## Typography

All sizes relative to Payload's admin panel base. The custom nav uses slightly larger text than Payload's default nav (CONTEXT decision: "groessere Texte").

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Nav Link | 14px | 400 (regular) | 1.4 | All clickable nav items in inactive state (direct links + dropdown items) |
| Nav Link Active / Section Header | 14px | 600 (semibold) | 1.4 | Currently active nav link, dropdown toggle labels (Bestellungsverwaltung, Produktverwaltung, etc.) |
| Subgroup Heading | 11px | 600 (semibold) | 1.2 | Uppercase category labels (HAUPTPRODUKTE, AUSSTATTUNG, etc.) |

Notes:
- Payload's default nav uses ~13px for links. 14px is the decided "slightly larger" step.
- Only 2 weights in the entire nav: 400 (inactive links) and 600 (active links + section headers + subgroup headings).
- Active link state is communicated by triple signal (accent bar + background + font weight), so the weight change from 400 to 600 reinforces rather than solely carries the active state.
- Subgroup headings at 11px + uppercase + semibold creates the standard "overline" label pattern.
- Letter-spacing for subgroup headings: 0.05em (aids readability at small size with uppercase).

## Color

This phase operates inside Payload's admin panel which has its own theming (light/dark mode via `--theme-*` CSS variables). All colors MUST reference Payload's CSS variables or shadcn CSS variables to ensure both light and dark theme compatibility.

| Role | Value | Usage |
|------|-------|-------|
| Nav Background | `var(--theme-elevation-0)` | Sidebar background — matches Payload's admin sidebar |
| Nav Link Text (inactive) | `var(--theme-elevation-600)` | Dimmed but readable link text |
| Nav Link Text (active) | `var(--theme-elevation-900)` | Full contrast active link text |
| Nav Link Text (hover) | `var(--theme-elevation-800)` | Slightly brighter on hover |
| Active Background | `var(--theme-elevation-100)` | Subtle background highlight on active link |
| Active Accent Bar | `var(--theme-success-500)` | Left border accent — uses Payload's success green for "you are here" semantics |
| Hover Background | `var(--theme-elevation-50)` | Subtle background on hover (lighter than active) |
| Section Header Text | `var(--theme-elevation-800)` | Dropdown toggle text — slightly less than full contrast |
| Subgroup Heading Text | `var(--theme-elevation-450)` | Muted uppercase category labels |
| Badge Destructive BG | shadcn `destructive` variant | Error count badge background (red) |
| Badge Destructive Text | shadcn `destructive-foreground` variant | Error count badge text (white) |
| Chevron Icon | `var(--theme-elevation-400)` | Dropdown toggle icon — subtle |

60/30/10 Split (within sidebar context):
- 60% Dominant: `var(--theme-elevation-0)` — sidebar background surface
- 30% Secondary: `var(--theme-elevation-50)` through `var(--theme-elevation-100)` — hover and active backgrounds
- 10% Accent: `var(--theme-success-500)` — reserved ONLY for the active link left accent bar

Accent reserved for: Active nav link left border (3px bar). Nothing else in the nav uses accent color.

## Interaction States

### Nav Link States

| State | Visual Treatment |
|-------|-----------------|
| Default (inactive) | Text: `elevation-600`, Weight: 400, no background, `cursor: pointer` |
| Hover | Text: `elevation-800`, Background: `elevation-50`, transition 150ms |
| Active (current page) | Text: `elevation-900`, Weight: 600, Background: `elevation-100`, Left border: 3px solid `success-500` |
| Active + Hover | Same as Active (no additional hover effect on already-active link) |

### Dropdown Section States

| State | Visual Treatment |
|-------|-----------------|
| Closed | Header text visible, ChevronDown rotated 0deg, content hidden |
| Open | Header text visible, ChevronDown rotated 180deg, content visible |
| Transition | ChevronDown rotation: `transition-transform duration-200` |

### WebhookFehlerBadge States

| State | Visual Treatment |
|-------|-----------------|
| No errors (count = 0) | Badge hidden (both locations) |
| Errors present (count > 0) | Shadcn Badge `variant="destructive"` showing count number |
| Fetch error / loading | Badge hidden (fail silently, show nothing) |

### Visual Hierarchy

1. **Direct links** — full-width clickable area, 36px min-height (hardcoded), 16px horizontal padding, 14px weight 400
2. **Section headers** — full-width clickable toggle, 36px min-height (hardcoded), 16px horizontal padding, 14px weight 600, ChevronDown icon right-aligned
3. **Dropdown items** — indented 16px from section header, 14px weight 400, 36px min-height (hardcoded)
4. **Subgroup headings** — indented same as dropdown items, 11px weight 600 uppercase, NOT clickable, 24px min-height
5. **Subgroup items** — indented 32px from section header (additional 16px from subgroup heading), 14px weight 400

## Accessibility Notes

- All nav links are `<a>` elements (via Next.js `<Link>`) with visible text labels
- Dropdown sections use Radix Collapsible which provides `aria-expanded` and `aria-controls` automatically
- Subgroup headings use `<span>` with `role="presentation"` — they are decorative category labels
- Active link additionally uses `aria-current="page"` attribute
- Minimum touch target: 36px height for all interactive elements (hardcoded CSS `min-h-[36px]`, not a spacing token — justified by WCAG 2.2 Target Size minimum)
- Color contrast: All text colors meet WCAG AA against their backgrounds (Payload's elevation scale is designed for this)
- No color-only state communication: active link uses accent bar + background + font weight (triple signal)
