# Milestone v1.2: Admin-Navigation Umbau

**Status:** SHIPPED 2026-03-23
**Phases:** 15-16
**Total Plans:** 4

## Overview

Custom Navigation Component fuer Payload Admin-Sidebar -- Standard-Nav komplett ersetzt durch aufklappbare Dropdowns mit Untergruppen, rollenbasierter Sichtbarkeit, session-persistentem Zustand und Kunden-Admin-Block.

## Phases

### Phase 15: Core Navigation

**Goal**: Admin sieht eine vollstaendig custom Sidebar mit festen Links, aufklappbaren Dropdowns, grauen Untergruppen-Ueberschriften und eingebettetem WebhookFehlerBadge -- die Standard-Payload-Nav ist komplett ersetzt
**Depends on**: Phase 14 (v1.1 abgeschlossen)
**Requirements**: NAV-01, NAV-02, NAV-03, NAV-04, NAV-05, NAV-06, NAV-07, NAV-08, UX-01, INT-01, INT-02
**Plans**: 2 plans

Plans:
- [x] 15-01: Shadcn-Installation, WebhookFehlerBadge Server-zu-Client Refactoring, Test-Infrastruktur
- [x] 15-02: Custom Nav Component, Config-Wiring, Visuelle Verifikation

**Details:**
- Shadcn Collapsible, Badge, Button installiert
- WebhookFehlerBadge von Server zu Client Component refactored (useEffect + fetch + 60s Polling)
- custom-nav.tsx: 4 Direct Links + 4 Dropdown-Sektionen + graue Untergruppen-Ueberschriften
- Inline Styles + scoped CSS (Payload Admin nutzt kein Tailwind)
- Active Link: Pill-Style mit rounded Background (dark-theme Referenz)
- Doppel-Badge: WebhookFehlerBadge bei System Header UND Webhook Fehler Link
- 31 Tests (20 custom-nav, 4 config, 7 badge)

### Phase 16: Session Persistence + Role Visibility

**Goal**: Dropdown-Auf/Zu-Zustand bleibt waehrend der Browser-Session erhalten und Nav-Links werden basierend auf der User-Rolle gefiltert
**Depends on**: Phase 15
**Requirements**: UX-02, UX-03
**Plans**: 2 plans

Plans:
- [x] 16-01: Session-Persistenz (sessionStorage dual-logic) und Rollen-basierte Nav-Filterung
- [x] 16-02: Kunden-Admin-Block (access.admin auf Users Collection) und Middleware-Redirect

**Details:**
- sessionStorage dual-logic: useRef prevPathnameRef unterscheidet Reload von SPA-Navigation
- Additive Opening: SPA-Navigation oeffnet aktive Sektion ohne andere zu schliessen
- filterByRole<T>: generische Funktion fuer rollenbasierte Array-Filterung
- access.admin auf Users Collection blockt Kunden server-seitig
- Middleware Redirect: Kunden werden von /admin zu /kunden/dashboard redirected
- 25 neue Tests (5 Session, 7 Role, 6 Access, 7 Middleware)

## Milestone Summary

**Key Decisions:**
- Custom DropdownSection statt NavGroup Override (NavGroup an usePreferences DB-Persistenz gekoppelt)
- Inline Styles statt Tailwind im Payload Admin Panel
- Dual-Layer Kunden-Block: access.admin + Middleware Redirect (Defense in Depth)
- WebhookFehlerBadge embedded statt afterNavLinks (Slot wird von custom Nav ueberschrieben)
- tsconfig.jest.json fuer JSX Transform Override in Jest

**Issues Resolved:**
- WebhookFehlerBadge Server/Client Boundary (refactored zu Client Component)
- Jest JSX Transform mit jsx:preserve (tsconfig.jest.json)
- Produktverwaltung Test: 17 Collections statt 16 (Zusatzlichter fehlte)

**Issues Deferred:**
- NAV-P2-01: Auto-Expand aktive Dropdown-Sektion
- NAV-P2-02: Animierte Dropdown-Uebergaenge (AnimateHeight)
- NAV-P2-03: Cross-Session Persistenz via usePreferences (DB-backed)

**Technical Debt Incurred:**
- hardcoded `/admin` prefix statt useConfig (low risk)
- Active link pill statt UI-SPEC accent bar (user-approved deviation)
- Test description mismatch in test-role-visibility.test.tsx
