# Requirements: Christ Fensterhandel Konfigurator

**Defined:** 2026-03-09
**Core Value:** Kunden konfigurieren Fenster/Tueren Schritt fuer Schritt mit intelligenter Filterung — Christ bekommt strukturierte Anfragen ins Dashboard

## v1 Requirements

### Setup

- [x] **SETUP-01**: Next.js App Router Projekt mit Payload CMS 3.0 embedded und PostgreSQL Datenbank
- [x] **SETUP-02**: Tailwind CSS + Shadcn UI Basis-Setup mit Design Tokens und Theme
- [x] **SETUP-03**: TypeScript strict, ESLint, saubere Ordnerstruktur, Git-ready

### CMS Collections

- [x] **CMS-01**: Collection `produkttypen` (Fenster, Balkontuer) mit Slug, Bild, erlaubte_materialien
- [x] **CMS-02**: Collection `materialien` (Kunststoff, Holz, Alu, K-Alu) mit Lieferzeit, Garantie, erlaubte_profile
- [x] **CMS-03**: Collection `profile` (Iglo 5, Iglo Energy, etc.) mit Uw-Wert, Kammern, Bautiefe, Min/Max Masse, material-Beziehung
- [x] **CMS-04**: Collection `fluegelanzahl` (1/2/3) mit fuer_produkttypen Beziehung
- [x] **CMS-05**: Collection `zusatzlichter` (Oberlicht, Unterlicht) mit kombinierbar_mit Beziehung
- [x] **CMS-06**: Collection `oeffnungsarten` (Fest, Kipp, Dreh, Dreh-Kipp) mit fuer_fenster/fuer_balkontuer
- [x] **CMS-07**: Collection `fensterformen` (Rechteck, Rundbogen, etc.) mit erlaubte_fluegelanzahl/oeffnungsarten
- [x] **CMS-08**: Collection `farben` mit Kategorie, RAL-Code, fuer_aussen/innen, erlaubte_materialien
- [x] **CMS-09**: Collection `dichtungsfarben`
- [x] **CMS-10**: Collection `verglasungen` (2-fach, 3-fach) mit Uw-Wert, Aufpreis
- [x] **CMS-11**: Collections `schallschutz`, `sicherheitsglas`, `glasdekore`, `sprossen`, `extras`
- [x] **CMS-12**: Collection `preisregeln` mit produkttyp/material/profil-Beziehung und Grundpreis pro m2
- [x] **CMS-13**: Collection `rabattcodes` mit Code, Typ (prozent/fest), Gueltigkeit, Min-Bestellwert
- [x] **CMS-14**: Collection `anfragen` mit Status-Workflow, Produkt-Array, Kontaktdaten, Snapshot
- [x] **CMS-15**: Collection `status_historie` (Audit Trail, unveraenderlich)
- [x] **CMS-16**: Konditionale API-Filterung funktioniert (Material → Profile, Produkttyp → Fluegel, etc.)
- [x] **CMS-17**: Seed-Script mit realistischen Testdaten (Drutex Iglo-Serie, Farben, Preise)

### Konfigurator

- [x] **KONF-01**: Landing Page mit 3 Konfigurator-Karten (Fenster/Balkontuer, Tueren, Rolllaeden)
- [x] **KONF-02**: Konfigurator-Routing: `/konfigurator/fenster`, `/konfigurator/tueren`, `/konfigurator/rolllaeden`
- [x] **KONF-03**: Step 1 — Produkttyp-Auswahl (Fenster/Balkontuer) mit Bild und Beschreibung
- [x] **KONF-04**: Step 2 — Material-Auswahl mit Badges (Lieferzeit, Garantie, Beliebtheit)
- [x] **KONF-05**: Step 3 — Profil-Auswahl (gefiltert nach Material) mit technischen Daten
- [x] **KONF-06**: Step 4 — Fluegel-Auswahl (gefiltert nach Produkttyp) + optionales Ober-/Unterlicht
- [x] **KONF-07**: Step 5 — Oeffnungsart pro Fluegel (gefiltert nach Produkttyp), Griff-Seite bei Dreh/Dreh-Kipp
- [x] **KONF-08**: Step 6 — Fensterform-Auswahl mit konditionalen Einschraenkungen
- [x] **KONF-09**: Step 7 — Masseingabe (Breite x Hoehe mm) mit Min/Max-Validierung vom Profil, Live-Vorschau
- [x] **KONF-10**: Step 8 — Farben (Aussen/Innen/Dichtung) mit "Gleich wie Aussen"-Option
- [x] **KONF-11**: Step 9 — Verglasung, Schallschutz, Sicherheitsglas, Glasdekor, Sprossen, Extras
- [x] **KONF-12**: Step 10 — Zusammenfassung mit Konfigurations-Uebersicht und Preisvorschau
- [x] **KONF-13**: Zurueck-Reset-Logik (Material aendern → Profil wird zurueckgesetzt)
- [x] **KONF-14**: Fortschrittsanzeige (StepIndicator) zeigt aktuellen Step
- [x] **KONF-15**: React Hook Form + Zod fuer alle Formular-Steps

### Warenkorb

- [x] **CART-01**: User kann konfiguriertes Produkt zum Warenkorb hinzufuegen
- [x] **CART-02**: User kann Stueckzahl pro Produkt aendern
- [x] **CART-03**: User kann Produkt bearbeiten (zurueck in Konfigurator mit Daten)
- [x] **CART-04**: User kann Produkt aus Warenkorb loeschen
- [x] **CART-05**: User kann weiteres Produkt hinzufuegen (neuer Konfigurator-Durchlauf)
- [x] **CART-06**: Warenkorb zeigt Zwischensumme aller Produkte

### Preisberechnung

- [x] **PREIS-01**: Client-seitige Preisvorschau (Flaeche * Grundpreis + Aufpreise)
- [x] **PREIS-02**: Server-seitige verbindliche Preisberechnung (API Route mit Zod-Validierung)
- [x] **PREIS-03**: Preisregeln aus CMS (Grundpreis pro m2 nach Produkttyp/Material/Profil)
- [x] **PREIS-04**: Rabattcode-Validierung (gueltig, abgelaufen, aufgebraucht, min-bestellwert)
- [x] **PREIS-05**: Konfigurations-Snapshot wird bei Absenden als JSON eingefroren

### Anfrage-Absenden

- [x] **SEND-01**: Kontaktformular (Vorname, Nachname, E-Mail, Telefon, Adresse, Nachricht, Datenschutz)
- [x] **SEND-02**: Server-seitige Validierung aller Kontaktdaten (Zod)
- [x] **SEND-03**: Anfrage wird in CMS gespeichert mit Status "NEU" und Snapshot
- [x] **SEND-04**: Weiterleitung zur Danke-Seite mit Bestaetigungs-Info

### Security

- [x] **SEC-01**: Server-seitige Zod-Validierung auf allen API-Endpoints
- [x] **SEC-02**: Access Control: Admin darf alles, Mitarbeiter bearbeitet Anfragen, Viewer nur lesen
- [x] **SEC-03**: Rollen-Feld in Users Collection (admin/mitarbeiter/viewer)

### Admin Dashboard

- [x] **ADMIN-01**: Anfragen-Liste mit Filter nach Status, Datum, Kategorie
- [x] **ADMIN-02**: Suchfunktion (Kundenname, E-Mail, Anfrage-Nr.)
- [x] **ADMIN-03**: Anfrage-Detailansicht mit allen Konfigurationsdaten
- [x] **ADMIN-04**: Status aendern (NEU → IN BEARBEITUNG → BESTAETIGT → BEZAHLT → ABGESCHLOSSEN + RUECKFRAGE/ABGELEHNT)
- [x] **ADMIN-05**: Status-Historie zeigt wer, wann, von/zu welchem Status (unveraenderlich)
- [x] **ADMIN-06**: Interne Notizen pro Anfrage (nur Admin/Mitarbeiter sichtbar)

### Kunden

- [x] **KUND-01**: Kunden-Collection mit auth: true (Login/Register)
- [x] **KUND-02**: Geschuetzte Route `/kunden/dashboard` (Redirect ohne Login)
- [x] **KUND-03**: Kunde sieht nur eigene Anfragen (Access Control serverseitig)
- [x] **KUND-04**: Status-Timeline zeigt Anfrage-Fortschritt chronologisch

### Stripe

- [x] **PAY-01**: Stripe Checkout Session erstellen aus Anfrage
- [x] **PAY-02**: Stripe Webhook empfangen und Status auf "BEZAHLT" setzen
- [x] **PAY-03**: Test-Modus mit Karte 4242 4242 4242 4242

### N8N Integration

- [x] **N8N-01**: Payload afterChange Hook sendet Webhook an N8N bei neuer Anfrage
- [x] **N8N-02**: Webhook-Security mit Secret/Token (ohne Secret → 401)
- [x] **N8N-03**: Formatierte E-Mail an Firma (Produktliste, Kundendaten)
- [x] **N8N-04**: Bestaetigungs-E-Mail an Kunde
- [x] **N8N-05**: Status-Aenderung triggert passende E-Mail

### Website / Page Builder

- [x] **WEB-01**: Puck Editor Plugin installiert und im Admin integriert
- [x] **WEB-02**: Mindestens 3 Puck Components (Hero, TextBlock, FeatureGrid)
- [x] **WEB-03**: Live Preview im Admin mit Breakpoints (Mobile/Tablet/Desktop)
- [x] **WEB-04**: Draft Mode (Entwurf vs. veroeffentlicht)

### Mehrsprachigkeit

- [x] **I18N-01**: Payload i18n konfiguriert fuer DE + EN
- [x] **I18N-02**: Lokalisierte Felder in CMS Collections (DE/EN Tabs)
- [x] **I18N-03**: Frontend-Sprachumschalter wechselt DE/EN

### DSGVO

- [x] **DSGVO-01**: Datenschutz-Checkbox ist Pflichtfeld bei Anfrage-Absenden
- [x] **DSGVO-02**: Cookie-Banner erscheint beim ersten Besuch
- [x] **DSGVO-03**: Datenloeschung im Admin moeglich

### Deployment

- [ ] **DEPL-01**: Dockerfile fuer Next.js + Payload App
- [ ] **DEPL-02**: PostgreSQL als Docker Container
- [ ] **DEPL-03**: Coolify-Deployment auf Netcup VPS mit SSL (Let's Encrypt)
- [ ] **DEPL-04**: N8N als Docker Container auf demselben Server
- [ ] **DEPL-05**: Auto-Deployment bei Git Push

- **ADMIN-V2-02**: Metriken-Dashboard mit Recharts (Anfragen/Monat, Kategorien)
- **ADMIN-V2-03**: Gast-Tracking (Anfrage-Nr + E-Mail → Status ohne Login)

### Integrations

- **INT-V2-01**: PDF-Generierung (Angebots-PDF aus Anfrage)
- **INT-V2-02**: Rate-Limiting auf API-Endpoints
- **INT-V2-03**: N8N Cron-Workflow fuer alte unbearbeitete Anfragen

### Konfigurator Erweiterungen

- **KONF-V2-01**: Tueren-Konfigurator (eigener Step-Flow)
- **KONF-V2-02**: Rolllaeden-Konfigurator (eigener Step-Flow)
- **KONF-V2-03**: Drutex API-Integration (automatische Bestellung)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Webflow-basierte Loesung | Komplett auf Next.js + Payload migriert |
| Supabase / externe DB | PostgreSQL direkt, kein externer Service |
| Retool/Softr Dashboard | Payload Admin Panel ist das Dashboard |
| Echtzeit-Chat | Nicht Teil des Konfigurators, hohe Komplexitaet |
| Mobile App | Web-first, responsive reicht fuer v1 |
| Garagentore/Raffstore | Eigene Konfiguratoren, spaeter |
