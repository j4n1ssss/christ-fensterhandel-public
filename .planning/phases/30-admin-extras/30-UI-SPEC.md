# Phase 30 — UI Design Contract

> Visual and interaction contract for Payload CMS Admin custom views. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Important context:** Phase 30 is entirely within the **Payload CMS Admin Panel**. The admin panel does NOT use Tailwind CSS or Shadcn UI. All styling uses inline styles + BEM CSS classes in `src/app/(payload)/custom.scss`, referencing Payload's built-in CSS custom properties (`--theme-elevation-*`, `--theme-bg`, etc.). This contract defines extensions to the established admin design system.

---

## Component Inventory

### 1. Webhook Tab (New Component: `webhook-tab.tsx`)

**Primary focal point:** The stats bar badges -- they communicate queue health at a glance and anchor the user's eye before scanning individual rows.

**Layout:**
```
+--------------------------------------------------+
| Stats Bar: "12 gesendet . 1 fehlgeschlagen . 0 abgebrochen" |
+--------------------------------------------------+
| Event Type     | Status | Zeitpunkt | Empfaenger |  <- compact row
|   [expand details when clicked]                   |
| Event Type     | Status | Zeitpunkt | Empfaenger |
|   Betreff: ...                                    |  <- expanded detail
|   Attempts: 3/5                                   |
|   Error: Connection timeout                       |
|   [Erneut versuchen]                              |
+--------------------------------------------------+
```

**Stats Bar:**
- Container: `display: flex; align-items: center; gap: 16px; padding: 8px 16px; background: var(--theme-elevation-50); border: 1px solid var(--theme-elevation-200); border-radius: 6px; margin-bottom: 16px`
- Each stat: inline badge with count + label
- Sent badge: `background: #22c55e20; color: #16a34a; padding: 4px 8px; border-radius: 12px; font-size: 12px; font-weight: 600`
- Failed badge: `background: #f9731620; color: #ea580c; padding: 4px 8px; border-radius: 12px; font-size: 12px; font-weight: 600`
- Dead badge: `background: #ef444420; color: #dc2626; padding: 4px 8px; border-radius: 12px; font-size: 12px; font-weight: 600`

**Queue Entry Row (collapsed):**
- Container: `display: flex; align-items: center; gap: 8px; padding: 8px 0; border-bottom: 1px solid var(--theme-elevation-150); cursor: pointer`
- Chevron icon: 12px, `var(--theme-elevation-500)`, rotates 90deg when expanded
- Event type: 13px weight 400, `var(--theme-elevation-800)`, truncate with ellipsis, max-width 200px
- Status badge: same pattern as `.status-badge` -- `padding: 2px 8px; border-radius: 12px; font-size: 11px; font-weight: 600; color: #fff` with background matching status color
- Timestamp: 12px weight 400, `var(--theme-elevation-500)`, relative format ("vor 2 Stunden")
- Empfaenger: 12px weight 400, `var(--theme-elevation-600)`, truncate with ellipsis

**Queue Entry Row (expanded):**

**Empty state:**

- Copy: "Keine E-Mail-Events fuer diese Anfrage."

**CSS class prefix:** `webhook-tab__` (BEM)

**Primary focal point:** The template dropdown and "E-Mail senden" submit button -- the dropdown initiates the compose flow and the submit button is the terminal action.

**Layout:**

| [x]                               |
+------------------------------------+
| Template:  [Dropdown v]           |

- Body: `.email-modal__body` -- padding 24px, overflow-y auto, flex 1
- Footer: `.email-modal__footer` -- sticky bottom, 16px vertical / 24px horizontal padding, border-top, flex space-between
- Cancel button: `.btn-ghost`
- Submit button: `.btn-primary` with `background: #3b82f6`

**Form sections:**
- Each section: `.email-modal__section` -- margin-bottom 16px
- Form labels: `.text-label` (12px weight 600)
- Template dropdown: `.form-select`
- Betreff input: `.form-input`
- Freitext textarea: `.form-textarea` rows=4

- Radio group: `.email-modal__radio-group` -- same pattern as `.refund-modal__radio-group` (flex column, gap 8px, margin-bottom 16px)
- Radio labels: 13px, `var(--theme-elevation-800)`, flex align-center gap 8px
- Alt empfaenger field: only visible when radio is not "default", `.form-input`

- Uses `srcdoc` attribute (not src) to avoid CSP issues

- Uses existing `.error-message` class or `.angebots-modal__error` pattern

**Primary focal point:** The filter tab bar with counts -- it orients the admin to their current view context and outstanding work volume before they scan row data.

- Fetches from `/api/admin/anfragen-list` API route
- Tab counts come from API response (no client-side counting)
- Pagination controls become server-driven

**Filter Tab Labels with Counts:**
- Format: `"Offen (12)"` -- count in parentheses, separated by space
- Active tab: existing `.list-tab__trigger--active` with `border-bottom-color: #3b82f6`
- Count rendered as: `font-weight: 400` even on active tab (only the label text is bold)

**Pagination Controls:**
- Left: "Seite X von Y" -- existing `.list-pagination__info` (12px, `var(--theme-elevation-500)`)
- Right: Previous/Next buttons using `.btn-ghost` + page number buttons
- Active page: `.btn-primary` style with `background: #3b82f6; color: #fff`
- Inactive pages: `.btn-ghost`
- Button size: `min-width: 32px; height: 32px; padding: 0 8px; font-size: 12px`
- Show max 7 page buttons with ellipsis (`...`) for large page counts

**Search input:**
- Existing `.list-search__input` + `.form-input` classes
- Placeholder: "Suche nach Nr., Name, E-Mail..."
- Debounce: 300ms before triggering API call
- Search icon (lucide Search, 14px) positioned inside input left with padding-left 32px

**Loading state during fetch:**
- Existing `.list-loading-state` (48px padding, centered, 13px)
- Copy: "Lade Anfragen..."

**URL State parameters:**
- `?page=2` -- current page (default: 1)
- `&tab=offen` -- active filter tab (default: alle)

## Interaction Patterns

### Webhook Tab Expand/Collapse
- Click anywhere on row to toggle expand
- Chevron rotates: 0deg (collapsed) to 90deg (expanded) with `transition: transform 0.15s ease`
- Only one row expanded at a time (accordion pattern) -- clicking another row collapses the previous
- Expanded area slides in with `max-height` transition (0 to auto via CSS or measured height via JS)

### Webhook Tab Retry
- Click "Erneut versuchen" button in expanded detail
- Button shows loading state: "Wird wiederholt..." with opacity 0.6
- On success: toast "E-Mail wurde erneut in die Queue eingereiht", row status updates to "pending" badge, stats bar re-counts
- On error: toast "Fehler beim Wiederholen", button returns to normal

### E-Mail Send Modal Open/Close
- Opens: click "E-Mail senden" button in Kontakt tab
- Closes: click overlay background, click X button, press Escape key
- Focus trap: focus stays within modal while open (tab cycling)
- On open: template dropdown has no selection, betreff empty, empfaenger pre-filled with customer email
- On close: form state reset (no persistence)

### E-Mail Send Modal Submit Flow
1. User selects template -> betreff auto-fills from Event-Matrix
2. User optionally edits betreff, adds freitext
3. User optionally changes empfaenger via radio + alt field
4. User clicks "Vorschau laden" -> iframe shows rendered HTML
5. User clicks "E-Mail senden" -> submit button shows "Wird gesendet..." with disabled state
6. Success: toast, modal closes, timeline refreshes (parent passes `refreshKey` increment)
7. Error: error message displayed in modal above form, modal stays open

### Pagination Navigation
- Click page number: fetches that page from API, updates URL params via `router.replace`
- Click Previous/Next: decrements/increments page, disables at boundaries
- Change filter tab: resets to page 1, fetches with new tab filter
- Change sort: resets to page 1, fetches with new sort
- Type in search: debounce 300ms, resets to page 1, fetches with search query
- Browser back/forward: restores full state from URL params

### Dashboard Refresh
- Dashboard converts from server component to client component
- Fetches `/api/admin/dashboard-stats` on mount with useEffect
- Loading state: all 5 stat cards show "..." as value while loading
- Error state: cards show "--" as value, no error banner (graceful degradation)
- No auto-refresh interval (manual page reload to update)
