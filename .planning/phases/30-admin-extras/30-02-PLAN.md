**1. Create WebhookTab component** (`src/components/admin/webhook-tab.tsx`):

"use client" component. Props: `{ anfrageId: string }`.

**Data fetching:**
- On mount, fetch `/api/email_queue?where[anfrage][equals]=${anfrageId}&sort=-createdAt&limit=50` with `credentials: "include"`
- Parse response: `data.docs` is the entry list
- Compute stats from docs: count entries by status (sent, failed, dead)

**Stats bar:**
- Container div with class `webhook-tab__stats`
- Three inline badges:
  - Sent: `{count} gesendet` with background `#22c55e20`, color `#16a34a`
  - Failed: `{count} fehlgeschlagen` with background `#f9731620`, color `#ea580c`
  - Dead: `{count} abgebrochen` with background `#ef444420`, color `#dc2626`
- Each badge: `padding: 4px 8px; border-radius: 12px; font-size: 12px; font-weight: 600`

**Queue entry rows:**
- State: `expandedId: string | null` (accordion -- only one open at a time)
- Each row: class `webhook-tab__row`, click toggles expand
- Collapsed content (flex row, gap 8px):
  - Chevron icon (ChevronRight from lucide-react, 12px) -- rotates 90deg when expanded via inline `transform: rotate(90deg)` and `transition: transform 0.15s ease`
  - Event type text (13px, truncate with `overflow: hidden; textOverflow: ellipsis; whiteSpace: nowrap; maxWidth: 200px`)
  - Status badge: pill with white text, background per status color (#22c55e=sent, #f97316=failed, #ef4444=dead, #3b82f6=pending), `padding: 2px 8px; border-radius: 12px; font-size: 11px; font-weight: 600`
  - Timestamp: `formatRelativeTime(entry.createdAt)` imported from `@/lib/list-view-helpers`
  - Empfaenger: entry.to (12px, truncate, elevation-600)

- Expanded detail (class `webhook-tab__detail`, only when expandedId matches):
  - Padding-left 24px, font-size 12px
  - Key-value pairs (label in elevation-500, value in elevation-800):
    - "Betreff:" entry.subject
    - "Empfaenger:" entry.to
    - "Versuche:" `${entry.attempts}/${entry.max_attempts}`
    - "Idempotency-Key:" entry.idempotency_key (truncated, 11px monospace)
  - Error log (only if entry.error_log): monospace block with `background: #ef444410; color: #ef4444; padding: 4px 8px; border-radius: 4px; white-space: pre-wrap; max-height: 80px; overflow-y: auto; font-size: 12px`
  - Retry button (only if status is "failed" or "dead"): class `btn-ghost`, text "Erneut versuchen"
    - onClick: PATCH `/api/email_queue/${entry.id}` with body `{ status: "pending", attempts: 0, next_retry_at: null, error_log: "" }`
    - On success: toast("E-Mail wurde erneut in die Queue eingereiht"), re-fetch data
    - On error: toast("Fehler beim Wiederholen")
    - Loading state: button text "Wird wiederholt...", opacity 0.6, disabled

**Empty state:**
- If docs.length === 0: div with class `webhook-tab__empty`, text "Keine E-Mail-Events fuer diese Anfrage."

**Loading state:**
- While fetching: "Lade E-Mail-Events..." centered text

**Betreff auto-fill on template change:**
Import EVENT_MATRIX from `@/lib/email/event-matrix`. When templateSlug changes (except "freitext"), look up `EVENT_MATRIX[templateSlug]?.betreff?.kunde` and use `renderSubject` pattern to fill in `#{anfrage_nummer}` with anfrageNummer. For "freitext", set subject to "".

Since EVENT_MATRIX keys use underscores (e.g., `neue_anfrage`) but template slugs use hyphens (e.g., `anfrage-bestaetigung`), and the EVENT_MATRIX maps to templates NOT the other way, iterate EVENT_MATRIX to find the entry whose `templates.kunde` matches the selected templateSlug, then use that entry's `betreff.kunde`.
