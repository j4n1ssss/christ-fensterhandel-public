    ```typescript
    import { NextRequest, NextResponse } from 'next/server';
    import { getPayload } from 'payload';
    import config from '@payload-config';
    import { TEMPLATE_SLUGS } from '@/lib/email/render-email';

    export async function GET(request: NextRequest) {
      const payload = await getPayload({ config });
      const { user } = await payload.auth({ headers: request.headers });
      if (!user || !['admin', 'mitarbeiter'].includes(user.rolle || '')) {
        return NextResponse.json({ error: 'Zugriff verweigert' }, { status: 403 });
      }

      const html = `<!DOCTYPE html>
      <html lang="de">
      <head><meta charset="UTF-8"><title>E-Mail Templates</title>
      <style>body{font-family:system-ui,sans-serif;max-width:600px;margin:0 auto;padding:32px}
      h1{font-size:24px;margin-bottom:24px}
      ul{list-style:none;padding:0}
      li{margin-bottom:8px}
      a{color:#3b82f6;text-decoration:none;font-size:16px}
      a:hover{text-decoration:underline}</style>
      </head>
      <body>
      <h1>E-Mail Templates</h1>
      <ul>${TEMPLATE_SLUGS.map(slug => `<li><a href="/api/email-preview/${slug}">${slug}</a></li>`).join('')}</ul>
      </body></html>`;

      return new NextResponse(html, { headers: { 'Content-Type': 'text/html; charset=utf-8' } });
    }
    ```

    ```markdown
    # N8N E-Mail Setup

    ## Uebersicht

    Die Anwendung verwendet N8N als reinen E-Mail-Transport. Die App rendert HTML-Templates,
    die E-Mail Queue speichert und versendet sie ueber N8N an den konfigurierten SMTP-Provider.

    ## Voraussetzungen

    - N8N Instanz (Coolify, Docker oder Cloud)
    - SMTP-Provider (z.B. Mailgun, Postmark, Amazon SES, eigener SMTP)
    - Zugang zur .env-Konfiguration der App

    ## N8N Workflow einrichten

    ### 1. Neuen Workflow erstellen

    1. In N8N neuen Workflow erstellen: "E-Mail Versand"
    2. Webhook Trigger hinzufuegen (POST)
    3. Webhook-URL kopieren

    ### 2. Workflow-Aufbau

    ```
    [Webhook Trigger] -> [Send Email Node] -> [Respond to Webhook]
    ```

    **Webhook Trigger:**
    - Methode: POST
    - Pfad: /email-send (oder auto-generiert)
    - Response: "Last Node"

    **Send Email Node (SMTP):**
    - Recipient: `{{ $json.to }}`
    - Subject: `{{ $json.subject }}`
    - HTML Body: `{{ $json.html }}`
    - Plain Text: `{{ $json.plain_text }}`
    - Reply-To: `{{ $json.reply_to }}`
    - From: Konfiguriert im SMTP-Credential

    **Respond to Webhook:**
    - Response Code: 200
    - Body: `{ "success": true }`

    ### 3. SMTP Credentials

    In N8N unter Credentials neues "SMTP" Credential anlegen:
    - Host: (Provider-abhaengig)
    - Port: 587 (TLS) oder 465 (SSL)
    - User: (Provider Login)
    - Password: (Provider API-Key oder Passwort)
    - SSL/TLS: An

    ### 4. Workflow aktivieren

    Workflow auf "Active" stellen.

    ## App-Konfiguration

    In `.env` die Webhook-URL eintragen:

    ```
    N8N_EMAIL_WEBHOOK_URL=https://n8n.example.com/webhook/xxxxx
    ```

    ## Testing-Checkliste

    - [ ] N8N Workflow ist aktiv
    - [ ] SMTP Credentials sind konfiguriert und getestet
    - [ ] `.env` enthaelt N8N_EMAIL_WEBHOOK_URL
    - [ ] Test-E-Mail ueber `/api/email-preview` versendet
    - [ ] E-Mail im Postfach angekommen
    - [ ] HTML-Darstellung korrekt (Logo, Footer, Buttons)
    - [ ] Plain-Text-Version lesbar
    - [ ] Reply-To Header korrekt
    - [ ] Queue-Eintrag Status = 'sent'

    ## Provider-Wechsel

    1. Neues SMTP Credential in N8N anlegen
    2. Send Email Node auf neues Credential umstellen
    3. Test-E-Mail versenden
    4. Alte Credentials loeschen

    Kein App-Deployment noetig -- N8N ist nur Transport.

    ## Fehlerbehebung

    | Problem | Loesung |
    |---------|---------|
    | Queue-Eintraege bleiben 'pending' | N8N_EMAIL_WEBHOOK_URL pruefen, N8N Workflow aktiv? |
    | Status 'dead' nach 5 Versuchen | N8N Logs pruefen, SMTP Credentials gueltig? |
    | Status 'skipped' | E-Mail-Adresse ungueltig, Kundendaten pruefen |
    | E-Mail im Spam | SPF/DKIM/DMARC DNS-Records pruefen |
    | Doppelte E-Mails | Idempotency-Key Constraint greift -- pruefen ob N8N Timeout |

    ## Webhook-Payload

    Die App sendet folgendes JSON an N8N:

    ```json
    {
      "to": "kunde@example.com",
      "subject": "Ihre Anfrage ANF-2026-001 ist eingegangen",
      "html": "<!DOCTYPE html>...",
      "plain_text": "Guten Tag Max Mustermann, ...",
      "reply_to": "info@christ-fensterhandel.de"
    }
    ```

    N8N muss nur die Felder an den SMTP-Provider weiterleiten.
    ```
