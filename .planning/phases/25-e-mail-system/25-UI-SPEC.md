# Phase 25 -- UI Design Contract

> Visual and interaction contract for the E-Mail-System phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Classification

**This phase is primarily backend/infrastructure.** The UI surface area is limited to:

1. **React Email Templates** -- JSX components rendered to HTML for email clients (NOT browser UI). These follow email-specific styling rules, not the project's Tailwind/shadcn system.
2. **Admin Settings: 5th "E-Mail" Tab** -- Extension of the existing Custom Admin Page at `/admin/einstellungen`, using established `custom.scss` patterns and Payload theme variables.
3. **Admin E-Mail Queue** -- Payload Collection list view (auto-generated by Payload Admin), with a custom Retry button component.
4. **Admin Email Preview Route** -- Server-rendered HTML pages at `/api/email-preview` (template index) and `/api/email-preview/[template]` (individual preview).

## Spacing Scale

### Email Templates (inline styles)

Email templates do NOT use Tailwind spacing tokens. All spacing is declared as inline CSS `px` values.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline text gaps |
| sm | 8px | Paragraph spacing within sections |
| md | 16px | Content padding (vertical), inter-section gaps |
| lg | 24px | Header/footer vertical padding |
| xl | 32px | Container horizontal padding, content horizontal padding |
| 2xl | 48px | Not used in email templates |

Exceptions: Email `<Container>` max-width is 600px (email client standard, not a spacing token).

### Admin UI (custom.scss)

Uses established project spacing from `custom.scss`. No new tokens needed.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Label-to-input gap (`margin-bottom: 4px`) |
| sm | 8px | Form group spacing, row gaps |
| md | 16px | Tab content padding, section gaps |
| lg | 24px | Page-level padding, section margins |

Exceptions: None -- Phase 25 admin UI follows existing `settings-page.tsx` patterns exactly.

## Color

### Email Templates (inline hex values)

Email clients do not reliably support CSS variables. All colors are declared as hex in inline styles.

| Role | Value | Usage |
|------|-------|-------|
| Background (outer) | `#f6f6f6` | Email body background (60%) |
| Background (content) | `#ffffff` | Content container (30%) |
| Text primary | `#1a1a1a` | Headings, body text |
| Text secondary | `#666666` | Footer text, secondary info |
| Text muted | `#999999` | Disclaimer text, timestamps |
| Accent / CTA | `#1a1a1a` | CTA Button background (10%) |
| CTA text | `#ffffff` | Button label text |
| Divider | `#e5e5e5` | Horizontal rules between sections |
| Destructive | `#ef4444` | Stornierung status, error states |
| Success | `#22c55e` | Bezahlt status, confirmation states |
| Warning | `#eab308` | Rueckfrage status, attention states |

**Accent reserved for:** CTA buttons only ("Anfrage ansehen", "Jetzt bezahlen", "Angebot ansehen"). All CTA buttons use dark background (`#1a1a1a`) with white text (`#ffffff`) -- consistent with the project's `--color-primary` being neutral/dark.

**Status colors in email status badges:** Reuse the same status color mapping from `status-config.ts` as inline hex values.

### Admin UI

Uses existing Payload theme variables (`--theme-elevation-*`, `--theme-bg`). No new color tokens. Queue status indicators reuse existing patterns:

| Status | Color | Existing pattern |
|--------|-------|-----------------|
| pending | `#3b82f6` (blue) | Same as `.link-email` |
| processing | `#eab308` (yellow) | Same as `.attention-bar--warn` |
| sent | `#22c55e` (green) | Same as `.score-bar__fill--green` |
| failed | `#f97316` (orange) | Same as `.score-bar__fill--orange` |
| dead | `#ef4444` (red) | Same as `.btn-destructive` |
| skipped | `var(--theme-elevation-500)` | Muted/grey |

## Interaction States

### Email Template Preview Page

| State | Visual |
|-------|--------|
| Default | Template rendered in centered container, HTML view |
| Plain-text toggle on | Monospace text in `<pre>` block, same container |
| Test sending | Button shows "Wird gesendet...", disabled, opacity 0.6 |
| Test sent | Button reverts, toast/inline success message in green |
| Test error | Inline error message below button, red text |
| Rate limited | Button disabled, tooltip/text shows wait message |

### Settings E-Mail Tab

| State | Visual |
|-------|--------|
| Loading | `Einstellungen werden geladen...` (existing pattern) |
| Editing | Form fields editable, Save button enabled |
| Saving | Save button shows `Wird gespeichert...`, disabled, opacity 0.6 |
| Saved | Toast: `Einstellungen gespeichert` (existing pattern) |
| Error | `.error-message` class below form (existing pattern) |

### Queue Retry Button

| State | Visual |
|-------|--------|
| Default (dead status) | `.btn-ghost` visible, enabled |
| Non-dead status | Button hidden or disabled |
| Retrying | Button text: `Wird wiederholt...`, disabled |
| Success | Toast: `E-Mail wurde erneut in die Queue eingereiht`, entry status updates to `pending` |
