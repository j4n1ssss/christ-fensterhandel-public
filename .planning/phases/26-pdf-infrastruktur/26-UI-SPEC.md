## Interaction Contracts

### Admin Dokumente Panel (right column of anfrage-detail-view)

1. Panel renders below existing tabs in the right column.
2. Shows a list of all Angebote + Rechnungen + Gutschriften for the current Anfrage.
3. Each row shows: icon (document type), filename, version/date meta, "Herunterladen" button.
4. "Angebot erstellen" button is shown for admin role only. Clicking triggers `POST /api/pdf/angebot/[anfrageId]`. During generation: button shows "Wird erstellt..." and is disabled. On success: list refreshes. On failure: `.error-message` appears below button.
5. Download button triggers `GET /api/pdf/angebot/[anfrageId]?id=[dokument-id]` (or rechnung/gutschrift equivalent). Browser receives `application/pdf` with `Content-Disposition: attachment`.
6. Panel uses inline `loadDoc()` refresh pattern (same as existing detail-view refresh).

### Rechnungen List View (Admin)

1. Filter tabs: "Alle" | "Rechnungen" | "Gutschriften" — reuses `.list-tabs` pattern.
2. Active tab persisted in URL param `?typ=rechnung|gutschrift` (same as existing Anfragen list `?tab=` pattern).
3. Table columns: Nummer | Typ | Anfrage | Betrag | Erstellt | Download.
4. "Download" column: icon button (`.btn-ghost`, 12px), triggers direct PDF download.
5. No row click navigation (immutable collection — no edit view needed).
6. Empty state uses `.list-empty-state` class.

### Angebote List View (Admin)

1. Filter tabs: "Alle" | "Entwurf" | "Versendet".
2. Table columns: Nummer | Version | Anfrage | Betrag | Gueltig bis | Status | Download.
3. Row click navigates to Payload default collection entry (read-only for non-entwurf).
4. Download button in last column.

### PDF Preview Route (/api/pdf-preview/[type])

1. Staff-only route (admin + mitarbeiter), returns 403 for viewer and kunde.
2. Response: `Content-Type: application/pdf` streamed inline (not attachment) for browser preview.
3. No UI component — browser's native PDF renderer handles display.
4. URL querystring: `?type=angebot|rechnung|gutschrift`.

### Kunden-Dashboard Dokumente Section

1. Rendered inside existing Anfrage detail view in kunden area.
2. Shows only documents belonging to the logged-in customer's Anfrage.
3. Download button: shadcn `<Button variant="outline" size="sm">` with `<Download />` icon (lucide-react, already available in project).
4. Access check: API route verifies `anfrage.kunde === currentUser.id` before streaming PDF.

## PDF Document Layout Contract

This section defines the @react-pdf/renderer layout — consumed by executor when building PDF templates.

### Shared: A4 dimensions, system Helvetica font

- Page: A4 (595pt x 842pt), all margins 40pt
- Header block: Logo (left, max 120pt wide) + Firmendaten (right-aligned, 10pt, grey #6b7280)
- Trennlinie nach Header: 1pt solid #e5e7eb, marginBottom 20pt

### Angebots-PDF specific layout

| Section | Details |
|---------|---------|
| Empfaenger-Block | Kundenname + Adresse, 11pt, left-aligned, marginBottom 24pt |
| Dokumenten-Meta | "Angebot [ANG-Nummer]" (16pt, weight 700) + "Datum: [date]" + "Gueltig bis: [date]" (11pt, grey) |
| Positions-Tabelle | Header row: light grey bg (#f9fafb), 10pt, weight 700, UPPERCASE. Data rows: 10pt, alternating white/#fafafa |
| Produkt-Beschreibung | Pos-Nr | Bezeichnung (mehrzeilig, alle Konfig-Details) | Menge | EP | GP |
| MwSt-Block | Netto-Summe + MwSt (Satz + Betrag) + Brutto — rechtsbündig, 10pt |
| Widerrufsbelehrung | Textblock 9pt, grey #4b5563, separator oben 1pt, vor Footer |
| Footer | Dreispaltig: Bankverbindung | Steuer/USt-IdNr | Kontakt + Seitenzahl (8pt, grey #9ca3af) |

### Rechnungs-PDF specific layout

Same as Angebot plus:
- "RECHNUNG" statt "ANGEBOT" als Dokumenten-Titel
- Zahlungsvermerk-Block (Zahlungsziel, IBAN, Verwendungszweck) nach MwSt-Block
- 10 UStG-Pflichtangaben alle vorhanden (Steuernummer aus Settings, fortlaufende RE-Nummer, MwSt-Ausweis)
- Kein Widerrufsbelehrung-Block

### Gutschrift-PDF specific layout

Same as Rechnung plus:
- "GUTSCHRIFT" als Titel, GS-Nummer
- Referenz-Block: "Gutschrift zur Rechnung [RE-Nummer] vom [Datum]" — 10pt, italic, grey
- Betrag negativ dargestellt (Erstattungsbetrag)
- Kein Zahlungsvermerk-Block
