# Phase 27 — UI Design Contract

> Visual and interaction contract for all frontend-facing UI in the Stripe End-to-End payment flow. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope:** This phase has UI in TWO rendering contexts:
1. **Admin Panel** (Payload CMS custom components) -- ZahlungsPanel, RefundModal, AttentionBar badge extension
2. **Frontend/Kunden** (Next.js App Router) -- Danke-Seite, StripePayButton refactor, Fehler-Seite

Each context has its own styling system. The contract specifies both.

## Component Inventory

### New Admin Components

#### 1. ZahlungsPanel (right column, below TabPanel)

- **Location:** Below DokumentePanel in the right column of detail-layout
- **Visibility:** Only when `status` is in `['zahlungslink_versendet', 'bezahlt', 'an_hersteller', 'hersteller_bestaetigt', 'hersteller_bestaetigt_mit_vorbehalt', 'in_produktion', 'hersteller_problem', 'versandbereit', 'geliefert', 'abgeschlossen', 'rueckerstattung_ausstehend', 'rueckerstattung_abgeschlossen']`
- **Container:** `.zahlungs-panel` class in custom.scss, follows `.dokumente-panel` pattern (margin-top: 24px, padding: 16px, border-radius: 8px, elevation-50 background, elevation-200 border)
- **Header:** `.text-heading` "Zahlung" + payment status badge (inline style, pattern from attention-bar)
- **Content rows:** Key-value pairs using `.detail-field__label` + `.detail-field__value` pattern
  - Zahlungslink: truncated URL + copy-to-clipboard icon button (`.btn-ghost` style, 16px icon)
  - Ablaufdatum: formatted date-time (de-DE format)
  - Betrag: formatted with `formatCents()`
- **Collapsible details:** Toggle link "Details anzeigen" / "Details ausblenden" (`.link-edit` style)
  - Session-ID: monospace text
  - Payment-Intent-ID: monospace text
  - Stripe Customer: clickable link to Stripe Dashboard (`.link-email` style, opens new tab)
- **Actions:**
  - "Neuen Link erstellen" button (`.btn-primary` with `background: #3b82f6`): visible only when `stripe_payment_status === 'abgelaufen'`
  - "Rueckerstatten" button (`.btn-destructive`): visible only for admin role, only at post-bezahlt statuses
- **Already refunded display:** When `stripe_refunded_amount_cents > 0`: show "Bereits erstattet: X EUR" in muted text below the Betrag row

#### 2. RefundModal (Radix Dialog in admin context)

- **Trigger:** "Rueckerstatten" button in ZahlungsPanel
- **Overlay:** `position: fixed`, `background: rgba(0,0,0,0.5)`, `z-index: 100`
- **Dialog box:** `max-width: 480px`, centered, `background: var(--theme-bg)`, `border-radius: 8px`, `padding: 24px`, `box-shadow: 0 8px 32px rgba(0,0,0,0.15)`
- **Title:** `.text-display` size (20px/600) "Rueckerstattung"
- **Content:**
  - Already refunded info (if partial): `.error-message` style "Bereits erstattet: X EUR -- Verbleibend: Y EUR"
  - Radio group: "Volle Rueckerstattung (X EUR)" / "Teilweise Rueckerstattung"
  - Teilbetrag input: `.form-input` -- visible only when partial selected, validates `> 0` and `<= remaining`
  - Begruendung textarea: `.form-textarea`, `min-height: 80px`, required, `placeholder: "Grund fuer die Rueckerstattung..."`
- **Actions row:** `.btn-row` (flex end, gap 8px)
  - "Erstattung verwerfen" button: `.btn-ghost`
  - "Rueckerstatten" button: `.btn-destructive`, disabled until form valid
- **Confirm dialog:** Browser `window.confirm("X EUR wirklich rueckerstatten? NICHT rueckgaengig machbar!")` -- no custom dialog needed, consistent with existing anonymize pattern

#### 3. AttentionBar Payment Badge Extension

- **New element:** Payment status badge in `.attention-bar__left`, positioned after the status badge
- **Condition:** Only when `stripe_payment_status` exists and is not null
- **Style:** Same `.status-badge` pattern with payment-specific colors (see Color section above)
- **Copy icon:** Small clipboard icon (lucide `Copy`, 14px) next to badge when `stripe_checkout_url` exists, copies URL to clipboard on click
- **Toast feedback:** Use Payload's `toast.success("Zahlungslink kopiert")` on copy

### Modified Admin Components

#### 4. Splitbutton Preis-Guard

- **Change:** When transitioning to `zahlungslink_versendet`, check if `gesamtpreis > 0`
- **Block behavior:** If `gesamtpreis === 0` or `gesamtpreis === null`, show `toast.error("Kein Preis vorhanden. Zahlungslink kann nicht erstellt werden.")`
- **No visual change** to the Splitbutton itself

### New Frontend/Kunden Components

#### 5. Danke-Seite (/zahlung/[status]/page.tsx)

Three variants controlled by URL param: `erfolgreich`, `abgebrochen`, `fehler`

**Layout:** Centered content card, max-width 480px, vertically centered in viewport (min-height: calc(100vh - header))

**Variant: erfolgreich**
- Polling phase 1 (0-30s, payment processing):
  - Spinner: `animate-spin` on a circular icon (lucide `Loader2`, 32px, `text-muted-foreground`)
  - Text: `text-lg font-semibold text-foreground` "Zahlung wird verarbeitet..."
  - Subtext: `text-sm text-muted-foreground` "Bitte warten Sie einen Moment."
- Polling phase 2 (payment confirmed):
  - Success banner: `rounded-xl border border-green-200 bg-green-50 p-6`
  - Checkmark icon: lucide `CheckCircle2`, 48px, `text-green-600`
  - Text: `text-xl font-bold text-green-800` "Zahlung erfolgreich!"
  - Subtext: `text-sm text-green-700` "Ihre Rechnung wird erstellt und per E-Mail zugesendet."
  - If logged in: `<Button>` (shadcn) "Zurueck zum Dashboard" linking to `/kunden/anfragen/{id}`
  - If not logged in: `text-sm text-muted-foreground` "Sie erhalten Ihre Rechnung in Kuerze per E-Mail."
- Polling phase 3 (30s timeout):
  - Info banner: `rounded-xl border border-blue-200 bg-blue-50 p-6`
  - Info icon: lucide `Info`, 48px, `text-blue-600`
  - Text: `text-xl font-bold text-blue-800` "Zahlung eingegangen!"
  - Subtext: `text-sm text-blue-700` "Die Verarbeitung kann einen Moment dauern. Ihre Rechnung erhalten Sie per E-Mail."
  - If logged in: same dashboard button as phase 2

**Variant: abgebrochen**
- Banner: `rounded-xl border border-orange-200 bg-orange-50 p-6`
- Icon: lucide `XCircle`, 48px, `text-orange-600`
- Text: `text-xl font-bold text-orange-800` "Zahlung abgebrochen"
- If logged in: `text-sm text-orange-700` "Sie koennen die Zahlung jederzeit erneut starten." + Button "Erneut bezahlen" linking to `/api/stripe/redirect/{anfrageId}`
- If not logged in: `text-sm text-orange-700` "Nutzen Sie den Zahlungslink in Ihrer E-Mail, um die Zahlung erneut zu starten."

**Variant: fehler**
- Banner: `rounded-xl border border-red-200 bg-red-50 p-6`
- Icon: lucide `AlertTriangle`, 48px, `text-red-600`
- Text: `text-xl font-bold text-red-800` "Zahlung fehlgeschlagen"
- Subtext: `text-sm text-red-700` "Diese Zahlungsseite ist nicht mehr gueltig. Bitte kontaktieren Sie uns fuer Hilfe."
- Contact hint: `text-sm text-muted-foreground` -- Firmendaten aus Settings or hardcoded fallback

**Meta:** `<meta name="robots" content="noindex" />` on all variants

#### 6. StripePayButton Refactor (Kunden Dashboard)

- **Status trigger change:** Show at `zahlungslink_versendet` (not `bestaetigt`)
- **Remove fetch logic:** Replace POST `/api/stripe/checkout` with `window.location.href = /api/stripe/redirect/{anfrageId}`
- **Container:** Keep existing style: `rounded-xl border border-green-200 bg-green-50 p-4`
- **Display data:**
  - Betrag: `text-sm font-medium text-green-800` showing `formatCents(gesamtpreis)`
  - Ablaufdatum: `text-sm text-green-700` "Gueltig bis {date}" (when stripe_expires_at available)
- **Button text states:**
  - Default: "Jetzt bezahlen ({price})"
  - Loading: "Weiterleitung zu Stripe..."
  - Expired: "Jetzt bezahlen ({price})" (same text -- redirect auto-regenerates)
- **At status bezahlt:** Replace entire section with green confirmation: `rounded-xl border border-green-200 bg-green-50 p-4` with checkmark icon + "Zahlung erhalten am {date}"
- **At refund status:** Replace with neutral info: `rounded-xl border border-border bg-card p-4` with "Ihre Zahlung wurde zurueckerstattet." + Gutschrift PDF download link (same pattern as Dokumente section)

## Interaction Patterns

### Polling (Danke-Seite)

| Property | Value |
|----------|-------|
| Endpoint | `GET /api/stripe/payment-status?session_id={cs_...}` |
| Interval | 2000ms |
| Timeout | 30000ms (15 polls) |
| Transitions | spinner (0-30s) -> success banner (if bezahlt) OR timeout message (if 30s reached) |
| Cleanup | `clearInterval` on unmount AND on terminal state |

### Copy to Clipboard (Admin ZahlungsPanel + AttentionBar)

| Property | Value |
|----------|-------|
| API | `navigator.clipboard.writeText(url)` |
| Feedback | `toast.success("Zahlungslink kopiert")` via Payload UI |
| Fallback | No fallback needed (admin is always modern browser) |

### Collapsible Details (Admin ZahlungsPanel)

| Property | Value |
|----------|-------|
| Default state | Collapsed |
| Toggle | `useState<boolean>(false)`, toggle button as `.link-edit` styled text |
| Animation | None (instant show/hide) |

### Refund Modal Flow

| Step | UI State |
|------|----------|
| 1. Click "Rueckerstatten" | Modal opens with pre-filled total amount |
| 2. Select full/partial | If partial: show amount input field |
| 3. Enter amount (partial only) | Validate > 0 and <= remaining |
| 4. Enter reason | Textarea, required, min 1 char |
| 5. Click "Rueckerstatten" | window.confirm with final amount |
| 6a. Confirm | POST /api/stripe/refund, show loading state on button, toast on success/error |
| 6b. Cancel confirm | Return to modal |
| 7. Success | Close modal, trigger doc refresh (`handleStatusChanged()`) |

### StripePayButton Navigation

| Property | Value |
|----------|-------|
| Action | `window.location.href = /api/stripe/redirect/{anfrageId}` |
| Loading state | Button disabled + "Weiterleitung zu Stripe..." text |
| No fetch | Direct navigation, no client-side API call |
