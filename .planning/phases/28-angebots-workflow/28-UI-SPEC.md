# Phase 28 -- UI Design Contract

> Visual and interaction contract for the Angebots-Workflow phase. Two distinct UI contexts: Payload Admin Panel (custom.scss + inline styles) and Frontend (Tailwind + shadcn CSS variables). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

This phase operates across two UI systems. Each new component follows the conventions of its context.

### Context A: Payload Admin Panel

| Property | Value |
|----------|-------|
| Tool | Payload Admin CSS (custom.scss) |
| Preset | BEM-style classes + Payload theme variables |
| Component library | Radix UI (via Payload) |
| Icon library | lucide-react (existing) |
| Font | system-ui, -apple-system, sans-serif (inherited from Payload) |

### Context B: Frontend (Kunden-Dashboard, Public Pages, Konfigurator)

| Property | Value |
|----------|-------|
| Tool | shadcn (components.json exists) |
| Preset | default, neutral base, CSS variables enabled |
| Component library | shadcn/ui + Tailwind utilities |
| Icon library | lucide-react (existing) |
| Font | system-ui, -apple-system, sans-serif |

## Spacing Scale

### Admin Panel (custom.scss)

All values use multiples of 4. Source: existing custom.scss patterns.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, radio label gap, caption margin-top |
| sm | 8px | Compact element spacing, form-row-grid gap, radio-group gap |
| md | 16px | Default element spacing, panel padding, section margin-bottom |
| lg | 24px | Modal padding, admin-content padding, panel margin-top |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |

Exceptions: Modal max-height uses 85vh (viewport unit, per CONTEXT.md decision).

### Frontend (Tailwind)

Standard Tailwind spacing scale restricted to the standard set {4, 8, 16, 24, 32, 48, 64} only. Source: existing kunden/anfrage-detail.tsx patterns.

| Token | Tailwind Class | Usage |
|-------|---------------|-------|
| xs | gap-1 / p-1 (4px) | Icon inline spacing |
| sm | gap-2 / p-2 (8px) | Compact spacing, confirm button vertical padding, PDF download icon gap |
| md | gap-4 / p-4 (16px) | Card padding, default gaps, CTA margin-top |
| lg | gap-6 (24px) | Section spacing |
| xl | gap-8 (32px) | Not used in this phase |
| 2xl | mt-12 (48px) | Footer margin-top |

Exceptions: Touch target minimum 44px for "Angebot annehmen" button (mobile usability).

## Interaction Contracts

### IC-01: Angebots-Erstellungs-Modal (Admin)

**Trigger:** Click "Angebot erstellen" in Splitbutton (at in_bearbeitung) OR click "+ Neues Angebot erstellen" in DokumentePanel (at any status).

**Layout:**
- Fixed overlay (`.angebots-modal__overlay`): `position: fixed; inset: 0; background: rgba(0,0,0,0.5); z-index: 100;`
- Dialog (`.angebots-modal__dialog`): `max-width: 640px; width: calc(100% - 32px); max-height: 85vh; overflow-y: auto; background: var(--theme-bg); border-radius: 8px; padding: 0; box-shadow: 0 8px 32px rgba(0,0,0,0.15);`
- Sticky header: Modal title + close button, `position: sticky; top: 0; z-index: 1; background: var(--theme-bg); padding: 24px 24px 16px; border-bottom: 1px solid var(--theme-elevation-200);`
- Scrollable body: `padding: 24px;`
- Sticky footer: Submit button, `position: sticky; bottom: 0; z-index: 1; background: var(--theme-bg); padding: 16px 24px; border-top: 1px solid var(--theme-elevation-200);`

**Sections (top to bottom):**

1. **Positionen (read-only):** Each product shown as condensed card using `.product-card` classes. Display: Produkttyp, Menge, Masse, Farben, Verglasung. No editing.

2. **Gesamtpreis (default, always visible):**
   - Label: `.text-label` "Angebotspreis (brutto)"
   - Input: `.form-input` type="number" step="0.01" -- editable, EUR display
   - Below input: `.text-body-muted` showing "Netto: {calcNetFromGross result} | MwSt: {brutto - netto}" -- live-updated on every keystroke
   - Pre-filled with sum of anfrage.produkte prices

3. **Einzelpreise (collapsible):**
   - Toggle: `<button>` with `.text-caption` "Einzelpreise anpassen" + chevron icon
   - When expanded: One `.form-input` per position, label shows position name + "x{menge}"
   - Each input: Brutto cents, Netto + MwSt shown as `.text-caption` below
   - On any Einzelpreis change: Gesamtpreis auto-recalculates (sum of all positions)
   - On Gesamtpreis manual change AFTER Einzelpreis edit: Gesamtpreis value takes precedence, stored as flat override

4. **Begruendung (conditional):**
   - Shown only when ANY price differs from the calculated anfrage.produkte sum
   - Label: `.text-label` "Begruendung fuer Preisanpassung" + red asterisk
   - Input: `.form-textarea` minHeight 80px, placeholder "Warum weicht der Preis ab?"
   - Required: form cannot submit without it when visible

5. **Gueltigkeit:**
   - Label: `.text-label` "Gueltigkeitsdauer"
   - Input: `.form-select` with options: 14 Tage, 30 Tage, 60 Tage, 90 Tage
   - Default: value from Settings Global (`angebots_gueltigkeit_tage`)
   - Below select: `.text-caption` showing computed date "Gueltig bis: {DD.MM.YYYY}"

6. **Freitext:**
   - Label: `.text-label` "Individuelle Hinweise (optional)"
   - Input: `.form-textarea` minHeight 80px, placeholder "Optionale Anmerkungen zum Angebot..."

**Footer:**
- Left: `.btn-ghost` "Erstellung abbrechen" (closes modal, no action)
- Right: `.btn-dokument-create` "Angebot erstellen und senden" (blue, #3b82f6)
- Disabled state: `opacity: 0.6; cursor: not-allowed;` when Begruendung required but empty OR submitting
- Loading state: "Wird erstellt..." text, button disabled

**Submit behavior:** Single API call. On success: modal closes, toast "Angebot erstellt und versendet", parent reloads data. On error: `.error-message` inside modal above footer.

**Pre-fill for new versions:** When creating V2+, modal pre-fills Gesamtpreis with last Angebot's betrag_brutto_cents. Version label shown in header: "Angebot V{n} erstellen".

### IC-02: Angebots-Annahme -- Public Route (/angebot/[anfrageId])

**Layout:** Full-width page with centered content container. Uses frontend Tailwind styles.

```
max-w-2xl mx-auto px-4 py-8 sm:py-12
```

**Sections:**

1. **Header:** Company logo (if available from Settings) + "Ihr Angebot" heading (`text-xl font-semibold`)

2. **Angebots-Zusammenfassung Card:**
   - `rounded-xl border border-border bg-card p-6`
   - Anfrage-Nummer: `text-sm text-muted-foreground`
   - Produkt-Liste: Each product as compact row (Name, Menge, Einzelpreis) using `text-sm`
   - Gesamtpreis: `text-lg font-semibold text-foreground` with separator line above (`border-t border-border pt-4 mt-4`)
   - Gueltig bis: `text-sm text-muted-foreground` with date

3. **PDF Download:**
   - `inline-flex items-center gap-2 rounded-md border border-border px-4 py-2 text-sm font-semibold text-foreground hover:bg-accent`
   - Label: "Angebot als PDF herunterladen"
   - Icon: `FileDown` from lucide-react, `h-4 w-4`

4. **Annahme-Bereich:**
   - When valid: "Angebot annehmen und zahlen" button (see IC-03 below)
   - When expired: Yellow/amber notice box: `rounded-xl border border-amber-200 bg-amber-50 p-4`
     - Text: "Dieses Angebot ist am {DD.MM.YYYY} abgelaufen."
     - Subtext: "Bitte kontaktieren Sie uns fuer ein neues Angebot: {telefon/email from Settings}"
     - Button disabled with `opacity-50 cursor-not-allowed`

5. **Footer:** Minimal company info from Settings (Firmenname, Telefon, E-Mail). `text-xs text-muted-foreground text-center mt-12`

### IC-03: Confirm-Dialog (before Stripe redirect)

**Trigger:** Click "Angebot annehmen und zahlen" on public page OR in Kunden-Dashboard.

**Implementation:** Inline expandable section (NOT a modal -- lighter pattern for single-page flow). Alternatively, a modal using Radix Dialog if imported. Decision: use inline section for simplicity.

**Layout:** Replaces the CTA button area with a confirmation section:

```
rounded-xl border border-border bg-card p-6
```

**Content:**

1. **Zusammenfassung:**
   - "Betrag: {formatPrice(gesamtpreis)}" -- `text-base font-semibold`
   - "inkl. {mwst_satz}% MwSt ({formatCents(mwst_cents)})" -- `text-sm text-muted-foreground`

2. **Widerrufs-Hinweis:**
   - `text-xs text-muted-foreground mt-4`
   - Copy: "Massanfertigungen sind vom Widerrufsrecht ausgenommen (Paragraph 312g Abs. 2 Nr. 1 BGB). Mit der Bestellung erklaeren Sie sich damit einverstanden, dass die Ware nach Ihren individuellen Vorgaben angefertigt wird."

3. **AGB-Checkbox:**
   - Same visual pattern as ContactForm datenschutz checkbox
   - Checkbox: `h-4 w-4 rounded border-border text-primary focus:ring-primary`
   - Label: `text-sm text-muted-foreground` -- "Ich akzeptiere die [AGB](dynamic link) und bestaetige die Bestellung."
   - AGB link: `text-primary underline underline-offset-2 hover:text-primary/80`
   - Validation: Required. Error shown as `text-xs text-red-600 ml-8` when form submitted without check.

4. **Buttons:**
   - Primary: `bg-primary text-primary-foreground px-6 py-2 text-sm font-semibold rounded-lg` -- "Verbindlich bestellen und zahlen"
   - Secondary: `text-sm text-muted-foreground underline cursor-pointer` -- "Annahme abbrechen" (collapses section, shows CTA button again)
   - Loading: "Weiterleitung zu Stripe..." with `cursor-wait` state

## State Mapping

### AngebotsModal States

| State | Visual | Interaction |
|-------|--------|-------------|
| Default (no price change) | Begruendung field hidden. Submit enabled. | Click submit -> API call |
| Price changed | Begruendung field appears with red asterisk. Submit disabled until filled. | Must enter reason |
| Submitting | Button shows "Wird erstellt...", all inputs disabled. | No interaction |
| Error | `.error-message` box above footer with error text. Inputs re-enabled. | Can retry |
| Success | Modal closes. Toast notification. Parent refreshes. | Automatic |

### Public Angebots-Page States

| State | Visual | Interaction |
|-------|--------|-------------|
| Valid angebot | Product summary + CTA button visible | Click CTA -> show confirm section |
| Confirm expanded | Confirm section replaces CTA. Widerrufshinweis + AGB checkbox + buttons. | Must check AGB to proceed |
| Redirecting to Stripe | "Weiterleitung zu Stripe..." loading state | No interaction |
| Expired angebot | Amber notice box. Button disabled. PDF download still active. | Can only download PDF |
| Not found / invalid | "Angebot nicht gefunden" with contact info | No interaction |

### AGB Checkbox States (Anfrage-Formular)

| State | Visual | Interaction |
|-------|--------|-------------|
| Unchecked | Empty checkbox, no error | Must check before submit |
| Checked | Filled checkbox | Form can submit |
| Validation error | Red error text below: "AGB muessen akzeptiert werden" | Must check to clear |
