**Part A: Create `src/components/admin/profile-last-editor.tsx`**

A "use client" component that renders in the `beforeDocumentControls` area alongside ProfileEditToolbar.

**Data access pattern (per user decision -- NO extra API call):** The `last_edited_by` field on the profile collection has `maxDepth: 1`, which means Payload already resolves the full user object into the admin form data when the document loads. Read the data directly from the form state using Payload admin hooks -- no fetch call needed.

Use `useAllFormFields()` from `@payloadcms/ui` to access the form state (same pattern as `use-undo-redo.ts`). Read `last_edited_by` and `updatedAt` field values from the returned `fields` FormState object:

```typescript
import { useAllFormFields, useDocumentInfo } from '@payloadcms/ui'

// Inside the component:
const [fields] = useAllFormFields()
const { id } = useDocumentInfo()

// Read the already-resolved last_edited_by value from form state.
// With maxDepth: 1, the value is the full user object (not just an ID string).
const lastEditedByField = fields?.['last_edited_by']
const lastEditedBy = lastEditedByField?.value as
  | { id: string; vorname?: string; nachname?: string; email: string }
  | string
  | null
  | undefined

const updatedAtField = fields?.['updatedAt']
const updatedAt = updatedAtField?.value as string | null | undefined
```

If `lastEditedBy` is a string (unresolved ID) or nullish, render nothing. If it is a resolved object, format and display. This approach follows the established codebase pattern (see `use-undo-redo.ts`) and honors the locked user decision: "Datenquelle: profile.last_edited_by + profile.updatedAt (kein extra API-Call)".

**Editor name formatting function:**
```typescript
function formatEditorName(editor: { vorname?: string; nachname?: string; email: string }): string {
  const parts = [editor.vorname, editor.nachname].filter(Boolean)
  const name = parts.join(' ')
  return name ? `${name} (${editor.email})` : editor.email
}
```

**Date formatting:**
```typescript
const dateFormatter = new Intl.DateTimeFormat('de-DE', {
  day: '2-digit', month: '2-digit', year: 'numeric',
  hour: '2-digit', minute: '2-digit',
})
```

**Layout -- renders inline with ProfileEditToolbar (both in beforeDocumentControls array):**
- Container: `display: flex`, `alignItems: center`, `gap: 8px`
- Vertical separator: `1px solid var(--theme-elevation-200)`, height `16px`, `margin: 0 8px`
- Text: `fontSize: 11px`, `fontWeight: 400`, `color: var(--theme-elevation-500)`
- Name portion: `fontWeight: 600`, `color: var(--theme-elevation-700)`
- Copy: "Zuletzt bearbeitet von **{Name}** ({email}) am {date}"

**States:**
- New document (no `id` from useDocumentInfo): return `null` (render nothing)
- Editor resolved with name: show full format
- Editor resolved without name (only email): show "Zuletzt bearbeitet von {email} am {date}"
- Editor relationship unresolved (ID string only, not object): return `null`
- No last_edited_by value (null/undefined): return `null`

**Part B: Create `src/components/admin/profile-history-panel.tsx`**

A "use client" component that serves as a Payload document view tab at route `/profile/{id}/history`.

**Data loading:**
On component mount, fetch from `/api/edit_history?where[collection][equals]=profile&where[doc_id][equals]={docId}&sort=-timestamp&limit=50&depth=1` with `credentials: 'include'`.

Use `useDocumentInfo()` from `@payloadcms/ui` to get the document ID.

**Response shape from REST API (with depth=1):**
```typescript
interface EditHistoryEntry {
  id: string
  collection: string
  doc_id: string
  event: string  // 'create' | 'update' | 'save_no_changes'
  diff: DiffEntry[] | null
  editor: { id: string; vorname?: string; nachname?: string; email: string } | string | null
  timestamp: string  // ISO date
}
```
