5. **computeDiff(previousDoc: Record<string, any>, doc: Record<string, any>): DiffEntry[]** -- pure function, no async:
   - Iterate all keys in the union of previousDoc and doc keys
   - Skip keys in EXCLUDED_FIELDS
   - For GROUP_FIELDS: iterate sub-keys and compare individually, using dot-path notation (e.g., `technische_daten.uw_wert`)
   - For RELATIONSHIP_FIELDS with hasMany (all erlaubte_* fields): normalize both to sorted string arrays, compare with JSON.stringify. If different, record `from` as the old array, `to` as the new array.
   - For RELATIONSHIP_FIELDS single (material): compare as strings directly.
   - For all other fields (text, number, select, checkbox, textarea, upload): compare with JSON.stringify. If different, record the change.
   - Return array of DiffEntry objects for all changed fields.
   - Handle null/undefined gracefully: treat null, undefined, and missing keys as equivalent for comparison (they all mean "no value").

6. **resolveRelationshipLabels(diff: DiffEntry[], payload: any): Promise<DiffEntry[]>** -- async, takes Payload instance:
   - For each DiffEntry whose `field` is in RELATIONSHIP_FIELDS:
     - Collect all unique IDs from both `from` and `to` values (handle both string for single relation and string[] for hasMany)
     - Call `payload.find({ collection, where: { id: { in: allIds } }, select: { [titleField]: true }, limit: allIds.length, depth: 0, pagination: false })`
     - Build ID-to-label map from results
     - Replace raw IDs with `{ id, label }` objects. For hasMany: `from` and `to` become `Array<{ id: string, label: string }>`. For single: `from` and `to` become `{ id: string, label: string } | null`.
   - Return the updated diff array (non-relationship entries pass through unchanged)
   - Wrap each collection lookup in try/catch -- if resolution fails, keep raw IDs (graceful degradation)

2. **profileAfterChange** -- Payload afterChange hook for profile collection:
```typescript
import type { CollectionAfterChangeHook } from 'payload'
import { computeDiff, resolveRelationshipLabels } from '@/lib/diff-utils'

export const profileAfterChange: CollectionAfterChangeHook = async ({
  doc,
  previousDoc,
  req,
  operation,
  context,
}) => {
  // Guard: prevent infinite loops
  if (context?.skipEditHistory) return

  try {
    let event: string
    let diff: any[] = []

    if (operation === 'create') {
      event = 'create'
      // No diff for create events -- just log the creation
    } else {
      // Compute diff between previousDoc and doc
      const rawDiff = computeDiff(previousDoc || {}, doc)

      if (rawDiff.length === 0) {
        event = 'save_no_changes'
      } else {
        event = 'update'
        // Resolve relationship labels (snapshot frozen at save time)
        diff = await resolveRelationshipLabels(rawDiff, req.payload)
      }
    }

    await req.payload.create({
      collection: 'edit_history',
      data: {
        collection: 'profile',
        doc_id: doc.id,
        event,
        diff: diff.length > 0 ? diff : null,
        editor: req.user?.id || null,
        timestamp: new Date().toISOString(),
      },
      overrideAccess: true,
      context: { skipEditHistory: true },
    })
  } catch (err) {
    // Non-blocking: log error but never prevent the profile save
    console.error('[Profile afterChange] Edit history error (non-blocking):', err)
  }
}
```
