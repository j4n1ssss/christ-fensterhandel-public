# Phase 24 -- UI Design Contract

> Visual and interaction contract for the Settings Custom Admin Page, Optimistic Locking toast, and Rate Limiting error responses. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

Phase 24 has a narrow UI surface. All UI lives inside the **Payload Admin Panel**, which uses its own theming system (`--theme-*` CSS variables), scoped SCSS (`custom.scss`), and inline styles. **No Tailwind in Admin.** No shadcn components in Admin.

| UI Element | Type | Location |
|------------|------|----------|
| Settings Custom Admin Page | New page, 4 tabs | `/admin/einstellungen` |
| Navigation link | Extend existing | System dropdown in `custom-nav.tsx` |
| Optimistic Locking toast | Toast notification | Anfrage detail view (existing) |
| Rate Limiting 429 response | JSON error body | API routes (no visual UI) |

## Color

The Settings page lives inside Payload Admin, which has its own dark/light theme system. All colors MUST use Payload theme variables, not hardcoded hex values.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--theme-bg)` | Page background, input backgrounds |
| Secondary (30%) | `var(--theme-elevation-50)` | Tab content panels, card backgrounds, form group backgrounds |
| Border | `var(--theme-elevation-200)` | Tab borders, input borders, card borders |
| Text primary | `var(--theme-elevation-900)` | Headings, active tab labels, field values |
| Text secondary | `var(--theme-elevation-600)` | Field labels, muted descriptions, inactive tabs |
| Text muted | `var(--theme-elevation-500)` | Placeholder text, captions, "last updated" info |
| Accent | `#3b82f6` | Active tab bottom border, links, "Einstellungen" nav link icon |
| Destructive | `#ef4444` | Error messages only (rate limit error, version conflict warning text) |
| Success | `#22c55e` | Save success toast background accent |
| Warning | `#eab308` | Version conflict toast background accent |

Accent reserved for: active tab indicator (bottom border), email/edit links, navigation icon highlight. NOT for buttons (buttons use `btn-primary` with dynamic inline background or `btn-save` with elevation colors).

**Source:** Extracted from existing `custom.scss`. Accent `#3b82f6` used in `.list-tab__trigger--active`, `.link-email`, `.link-edit`, `.terminal-info__reopen`. Destructive `#ef4444` used in `.btn-destructive`, `.error-message`, `.btn-anonymize`.

## Tab Content Specifications

### Tab 1: Firmendaten
- **Sections:** Firmenanschrift, Steuerdaten, Bankverbindung
- **Fields:** firmenname (text, required), strasse (text), hausnummer (text), plz (text), ort (text), telefon (text), email (email), steuernummer (text), ust_id (text), iban (text), bic (text), bank_name (text)
- **Note:** Bankverbindung section has a muted helper text "Optional -- Stripe ist primaeres Zahlungsmittel"

### Tab 2: Steuer
- **Sections:** MwSt-Konfiguration
- **Fields:** mwst_satz (number input, default 19, suffix "%"), preisanzeige (select: "Brutto (inkl. MwSt)" / "Netto (zzgl. MwSt)", default Brutto)
- **Helper text below mwst_satz:** "Wird fuer alle Preisberechnungen, PDFs und Stripe-Zahlungen verwendet"

### Tab 3: Stripe
- **Sections:** Zahlungseinstellungen
- **Fields:** zahlungslink_ablaufzeit_stunden (number input, default 24, suffix "Stunden"), waehrung (select: EUR/USD/PYG/etc., default EUR)
- **Info box:** "API-Keys werden ueber Umgebungsvariablen (.env) konfiguriert und sind hier aus Sicherheitsgruenden nicht sichtbar."
- **Info box style:** `.text-body-muted` with `padding: 12px 16px`, `background: var(--theme-elevation-50)`, `border: 1px solid var(--theme-elevation-200)`, `border-radius: 6px`

### Tab 4: Dokumente
- **Sections:** Angebots-Einstellungen, Rechtliches, PDF-Layout, E-Mail (vorbereitet)
- **Fields:** angebots_gueltigkeit_tage (number input, default 30, suffix "Tage"), widerrufsbelehrung (textarea, 6 rows), agb_link (text input, URL), agb_pdf (Payload upload/media relation), logo_pdf (Payload upload/media relation)
- **E-Mail subsection fields:** absender_name (text), reply_to (email), signatur (textarea, 4 rows)
- **E-Mail subsection helper text:** "Wird in Phase 25 (E-Mail-System) aktiviert"

## Interaction Specifications

### Settings Save Flow
1. User edits fields in any tab
2. User clicks "Einstellungen speichern" (`.btn-save`)
3. Button shows "Wird gespeichert..." with `opacity: 0.6` (disabled state, matches existing pattern)
4. On success: `toast.success('Einstellungen gespeichert')` via `@payloadcms/ui` toast
5. On error: `.error-message` element appears above the save button with specific validation error
6. "Zuletzt aktualisiert" timestamp refreshes to show current time + user name

### Tab Persistence
- Active tab stored in `sessionStorage` under key `settings-tab` (matches `anfrage-detail-tab` pattern from `tab-panel.tsx`)
- On page load: restore last active tab from sessionStorage, fallback to "Firmendaten"

### Optimistic Locking Toast (Anfrage Detail View)
- **Trigger:** API returns HTTP 409 Conflict when saving an Anfrage
- **Toast type:** `toast.error()` via `@payloadcms/ui`
- **Message:** "Diese Anfrage wurde zwischenzeitlich von einem anderen Benutzer geaendert. Bitte laden Sie die Seite neu."
- **Duration:** Persistent (no auto-dismiss) -- user must act
- **Action:** Page includes a "Seite neu laden" link/button that calls `window.location.reload()`

### Rate Limiting 429 Response (API Only)
- **No visual UI component.** The 429 response is a JSON body consumed by existing error handling.
- **Response body:** `{ "error": "Zu viele Anfragen. Bitte warten Sie einen Moment." }`
- **Headers:** `Retry-After: {seconds}`, `Content-Type: application/json`

### Navigation Update
- "Einstellungen" link added to the System dropdown in `custom-nav.tsx`
- Position: Below existing System items
- Icon: `Settings` from lucide-react (gear icon)
- Label: "Einstellungen" (no emoji, per project rules)
- Route: `/admin/einstellungen`

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Tab keyboard navigation | Arrow keys to switch tabs, Enter/Space to activate (standard tab pattern) |
| Form labels | Every input has a visible `<label>` element with `for` attribute |
| Required fields | Visual asterisk (*) after label text + `aria-required="true"` on input |
| Error association | Error messages linked via `aria-describedby` on the input |
| Toast announcements | Payload toast uses `aria-live="polite"` (built-in) |
| Focus management | After save error, focus moves to first invalid field |

## States Summary

| State | Visual |
|-------|--------|
| Loading | "Einstellungen werden geladen..." centered text (`.detail-view__loading` pattern) |
| Empty (first visit) | All fields empty with placeholder values, helper text visible |
| Filled | Fields populated from DB, "Zuletzt aktualisiert" shows timestamp |
| Saving | Button disabled + "Wird gespeichert..." text, `opacity: 0.6` |
| Save success | Toast notification, timestamp updates |
| Save error | `.error-message` element above save button |
| Read-only (non-admin) | All fields disabled, save button hidden, info text: "Nur Administratoren koennen Einstellungen aendern." |
| Version conflict (Anfrage) | Persistent error toast with reload action |
