# Phase 18 — UI Design Contract

> Visual and interaction contract for the status color system, admin form field groups, and customer-facing copywriting. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase nature:** Phase 18 is a data-model and server-side phase. No new frontend components are built. This contract locks down the **visual data** (colors, labels, copy) that status-config.ts will export, and the **admin form structure** (collapsible groups, field visibility) that anfragen.ts will configure. Phases 19-21 consume these values for rendering.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 18px | 600 (semibold) | 1.3 |
| Display | 24px | 600 (semibold) | 1.2 |

**Note:** 2 weights only -- 400 (regular) for Body and Label, 600 (semibold) for Heading and Display. Label at 12px is visually distinct from Body at 14px (2px gap). Display at 24px is sufficiently distinguished from Heading at 18px by its size step alone, so both share semibold weight. These values are reference for downstream phases. Phase 18 does not render new typography -- it writes string data (labels, customer texts) into status-config.ts.

## Admin Form Field Layout

### Collapsible Group: Hersteller-Informationen

- **Visibility condition:** Status is one of: `bezahlt`, `an_hersteller`, `hersteller_bestaetigt`, `hersteller_bestaetigt_mit_vorbehalt`, `in_produktion`, `hersteller_problem`, `versandbereit`, `geliefert`, `abgeschlossen`
- **Default state:** Collapsed (`initCollapsed: true`)
- **Field order within group:**
  1. `hersteller_bestellnummer` (text) -- first because it is the primary identifier
  2. `lieferdatum_erwartet` (date, format: dd.MM.yyyy) -- second because it is the key timeline info
  3. `hersteller_notizen` (textarea) -- admin's notes to/about manufacturer
  4. `hersteller_antwort` (textarea) -- manufacturer's response

### Collapsible Group: Stornierung

- **Visibility condition:** Status is `storniert`
- **Default state:** Expanded (`initCollapsed: false`) -- when visible, it is always relevant
- **Field order within group:**
  1. `stornierung_grund` (textarea, required) -- reason is always needed
  2. `rueckerstattung_betrag` (number, conditionally required) -- amount to refund
  3. `rueckerstattung_status` (select: ausstehend/durchgefuehrt/abgelehnt, conditionally required) -- tracking refund progress

### Standalone Field: last_status_change_at

- **Position:** After the `status` select field, before `produkte`
- **Display:** Read-only, format `dd.MM.yyyy HH:mm`
- **Always visible:** Yes (helps admin prioritize at a glance)
