## Spacing Scale

Declared values (multiples of 4 only). Source: established patterns in `angebots-annahme.tsx`, `login-form.tsx`, `status-banner.tsx`.

| Token | px value | Tailwind class | Usage |
|-------|----------|----------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gaps, inline padding, checkbox offset |
| sm | 8px | p-2 / gap-2 | Compact element spacing, button icon gap |
| md | 16px | p-4 / gap-4 | Default element spacing, banner padding |
| lg | 24px | p-6 / gap-6 | Card/section padding (login-form uses p-6) |
| xl | 32px | p-8 / gap-8 | Page-level component spacing |
| 2xl | 48px | py-12 | Auth page vertical centering |
| 3xl | 64px | — | Not used in this phase |

Exceptions:
- Touch targets for upload triggers: minimum 44px height (accessibility)
- Input fields and submit buttons: `py-3` (12px) — on-grid, preferred value for this phase
- Login-form cross-spec note: `src/components/kunden/login-form.tsx` currently uses `py-2.5` (10px, off-grid). That value is an existing component artifact and is NOT inherited into new components built in this phase. If visual consistency with login-form becomes a requirement, login-form.tsx must be updated to `py-3` as a separate task.
- Textarea: minimum `h-24` (96px) for comfortable text entry

## Typography

Source: `login-form.tsx`, `anfrage-detail` page, `angebots-annahme.tsx` patterns.

Two weights only.

| Role | Size | Tailwind | Weight | Line Height | Usage |
|------|------|----------|--------|-------------|-------|
| Body | 14px | text-sm | 400 (normal) | 1.5 | Form labels, field labels, helper text, textarea content, list items, banner text |
| Heading | 24px | text-2xl | 700 (bold) | 1.2 | Page headings (`font-bold`) — passwort-vergessen, reklamation page; also banner emphasis text (`font-bold`) |

Additional type rules:
- Error messages: `text-xs` (12px), weight 400, color `text-red-600`
- Muted/helper text: `text-sm text-muted-foreground` weight 400
- Field labels: `text-sm font-normal` (weight 400) — NOT `font-medium`
- Banner text (StatusBanner): `text-sm font-bold` (weight 700, aligns with heading weight)
- Inline confirm text (hint/disclaimer): `text-xs text-muted-foreground` weight 400
- Submit button labels: `text-sm font-normal` (weight 400) — visual prominence comes from `bg-primary`, not font weight

## Component Inventory

All components in this phase are new custom components, built on existing patterns. No new shadcn components need to be installed beyond what is already present (Badge, Button, Collapsible).

### Visual Hierarchy — Primary Screen (anfrage-detail-view)

The `StatusBanner` is the primary focal point of the anfrage-detail-view. It renders first in the content flow and communicates the current order state. All action CTAs (RueckfrageFormular trigger, StornoDialog trigger, ReklamationFormular trigger) are secondary anchors rendered below the StatusBanner in vertical order. No two action CTAs compete at the same visual level — only one is visible per status combination, with the exception of StornoDialog which co-exists as a discreet text link beneath the primary action.

### New Components

| Component | File | Type | Pattern Basis |
|-----------|------|------|---------------|
| RueckfrageFormular | `src/components/kunden/rueckfrage-formular.tsx` | Client | AngebotAnnahmeButton inline-expand pattern |
| StornoDialog | `src/components/kunden/storno-dialog.tsx` | Client | AngebotAnnahmeButton confirm-section pattern |
| ReklamationFormular | `src/components/kunden/reklamation-formular.tsx` | Client | RueckfrageFormular pattern |
| ReklamationAnzeige | `src/components/kunden/reklamation-anzeige.tsx` | Server | StatusBanner render pattern |

### Component Behavior Contracts

#### RueckfrageFormular (KUND-01)

Rendered as sibling to StatusBanner when `status === "rueckfrage"`.

**States:**
1. **Collapsed** — Shows "Jetzt antworten" button (`bg-primary text-primary-foreground`, `rounded-lg px-4 py-3 text-sm font-normal`)
2. **Expanded** — Inline form area within `rounded-xl border border-border bg-card p-4 mt-3`
   - Textarea: `min-h-24 w-full rounded-lg border border-border bg-background px-3 py-3 text-sm`, min 10 chars, required
   - File input: Simple `<input type="file">` wrapped in a styled label (`rounded-lg border-2 border-dashed border-border p-4 text-center`), not drag-and-drop. Accepts image/jpeg, image/png, image/webp, image/heic, application/pdf. Max 3 files, max 10MB each.
   - File list: Each selected file shown as `text-sm text-muted-foreground` with filename + size + remove X button (`h-4 w-4 text-muted-foreground hover:text-foreground`)
   - Submit button: `bg-primary text-primary-foreground rounded-lg px-4 py-3 text-sm font-normal`
   - Cancel affordance: `text-sm text-muted-foreground underline cursor-pointer` — collapses form, label: "Formular schliessen"
3. **Submitting** — Submit button shows `<Loader2 className="h-4 w-4 animate-spin" />` + "Wird gesendet...", `cursor-wait opacity-50`
4. **Submitted** — Form area replaced with success message inside `bg-cyan-50 border border-cyan-200 rounded-xl p-4`; text: "Ihre Antwort wurde erfolgreich uebermittelt. Wir melden uns." (`text-sm text-cyan-800`)
5. **Error** — Red error banner above submit button: `rounded-md bg-red-50 p-2 text-sm text-red-600`

#### StornoDialog (KUND-02)

Rendered at the bottom of anfrage-detail-view. "Stornierung beantragen" renders as a discreet text link, not a prominent button.

**Trigger:** `text-sm text-muted-foreground underline underline-offset-2 cursor-pointer hover:text-foreground` — NOT `bg-destructive`, not prominent.

**States:**
1. **Collapsed** — Link only: "Stornierung beantragen"
2. **Expanded** — Inline confirm section in `rounded-xl border border-border bg-card p-6 mt-4`
   - Hint text: `text-sm text-muted-foreground mb-4` — "Ihre Anfrage wird nicht automatisch storniert. Unser Team prueft Ihre Anfrage und meldet sich bei Ihnen."
   - Textarea: same pattern as RueckfrageFormular, placeholder: "Bitte beschreiben Sie den Grund fuer Ihre Stornierungsanfrage"
   - "Stornierung beantragen" submit: `bg-primary text-primary-foreground rounded-lg px-4 py-3 text-sm font-normal`
   - Cancel affordance: `text-sm text-muted-foreground underline cursor-pointer` — collapses form, label: "Anfrage behalten"
3. **Submitting** — Loading state same as RueckfrageFormular
4. **Submitted** — Link area disappears. StatusBanner updates to show amber `stornierung_beantragt` state.
5. **Error** — Red error banner as per pattern

#### ReklamationFormular (KUND-03)

Rendered after "Reklamation melden" button click. Only visible when `status === "geliefert" || status === "abgeschlossen"`.

**Trigger button:** `bg-primary text-primary-foreground rounded-lg px-4 py-3 text-sm font-normal inline-flex items-center gap-2` with AlertTriangle icon (`h-4 w-4`)

**States:**
1. **Collapsed** — Button only
2. **Expanded** — Inline form in `rounded-xl border border-border bg-card p-6 mt-4`
   - Beschreibung Textarea: min 20 chars, same pattern as above
   - Foto-Upload: same dashed-border file-input pattern, max 5 files, max 10MB each, images + PDF
   - Submit: "Reklamation einreichen" button
   - Cancel affordance: `text-sm text-muted-foreground underline cursor-pointer` — collapses form, label: "Formular schliessen"
3. **Submitted** — Form replaced with success state. `ReklamationAnzeige` takes over.

#### ReklamationAnzeige (KUND-03)

Static display section titled "Ihre Reklamation" in `rounded-xl border border-border bg-card p-6 mt-4`.

| Reklamation status | Background | Text |
|-------------------|------------|------|
| `offen` | `bg-red-50 border-red-200` | "Ihre Reklamation wird geprueft." |
| `in_bearbeitung` | `bg-amber-50 border-amber-200` | "Ihre Reklamation wird bearbeitet." |
| `geloest` | `bg-green-50 border-green-200` | "Ihre Reklamation wurde geloest." + Loesung-Text in `text-sm text-green-800` |

Photo thumbnails: `w-16 h-16 object-cover rounded-md border border-border` in a `flex flex-wrap gap-2` container. No lightbox (deferred). Clicking a thumbnail opens in a new browser tab via `<a href target="_blank">`.

#### PasswortVergessenForm + PasswortResetForm (KUND-04)

Match `LoginForm` pattern exactly, with `py-3` for inputs and submit buttons (this phase standard). If login-form.tsx uses `py-2.5`, that component is not modified here — see cross-spec exception note in Spacing Scale above.

- Container: `mx-auto w-full max-w-md` with `rounded-xl border border-border bg-card p-6 shadow-sm`
- Heading: `text-2xl font-bold text-foreground` centered
- Subtext: `mt-1 text-sm text-muted-foreground` centered
- Inputs: `w-full rounded-lg border border-border bg-background px-3 py-3 text-sm text-foreground placeholder:text-muted-foreground focus:border-primary focus:outline-none focus:ring-1 focus:ring-primary`
- Submit: `inline-flex w-full items-center justify-center gap-2 rounded-lg bg-primary px-6 py-3 text-sm font-normal text-primary-foreground transition-colors hover:bg-primary/90 disabled:opacity-50`
- Success state: `rounded-lg border border-green-200 bg-green-50 px-4 py-3 text-sm text-green-700`
- Error state: `rounded-lg border border-red-200 bg-red-50 px-4 py-3 text-sm text-red-700`

## Interaction Contracts

### Form Validation

All forms use React Hook Form + Zod. Validation is client-side-first, server confirms.

| Field | Rule | Inline error trigger |
|-------|------|---------------------|
| Rueckfrage Textarea | minLength 10 | On blur + on submit |
| Storno Textarea | minLength 10 | On blur + on submit |
| Reklamation Textarea | minLength 20 | On blur + on submit |
| Passwort neu | minLength 8 | On blur + on submit |
| Passwort bestaetigen | must match neu | On blur + on submit |
| E-Mail (passwort-vergessen) | valid email format | On blur + on submit |

Inline error placement: `mt-1 text-xs text-red-600` immediately below the field (matches login-form.tsx pattern).

### File Upload Constraints

| Constraint | Rueckfrage | Reklamation |
|------------|-----------|-------------|
| Max files | 3 | 5 |
| Max size per file | 10 MB | 10 MB |
| Accepted types | image/jpeg, image/png, image/webp, image/heic, application/pdf | image/jpeg, image/png, image/webp, image/heic, application/pdf |
| UI pattern | Simple file-input (no drag-and-drop) | Simple file-input (no drag-and-drop) |
| Pre-upload validation | Client-side size + type check before API call | Client-side size + type check before API call |
| Error: wrong type | "Nur Bilder (JPEG, PNG, WebP, HEIC) und PDF-Dateien erlaubt." | same |
| Error: too large | "Datei zu gross. Maximale Dateigroesse ist 10 MB." | same |
| Error: too many | "Maximal 3 Dateien erlaubt." | "Maximal 5 Dateien erlaubt." |

### Loading States

All async submit actions:
1. Disable submit button immediately on click
2. Show `<Loader2 className="h-4 w-4 animate-spin" />` in button
3. Replace button label with loading copy
4. Set `cursor-wait` on button

### Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Banner role | `role="status"` for informational, `role="alert"` for terminal/error (matches existing status-banner.tsx) |
| Form errors | `aria-invalid={true}` + `aria-describedby="{field}-error"` on invalid inputs |
| File input | Visible `<label>` wrapping `<input type="file">` — no hidden inputs |
| Loading announcement | `aria-live="polite"` on form container (matches angebots-annahme.tsx) |
| Disabled buttons | `disabled` + `aria-disabled="true"` |
| Touch targets | Minimum 44px height on all interactive elements |
