    - Query anfragen collection sorted by createdAt desc, where anfrage_nummer starts with current year prefix
    - Parse last number, increment by 1
    - If no existing anfragen for this year, start at 001
    - Handle retry on unique constraint failure (increment and retry once)

<task type="auto" tdd="true">
  <name>Task 2: API Routes (calculate-price + validate-discount) + discount tests</name>
  <files>src/app/api/anfrage/calculate-price/route.ts, src/app/api/anfrage/validate-discount/route.ts, tests/unit/test-discount-validation.test.ts, tests/unit/test-snapshot.test.ts</files>
  <behavior>
    - POST /api/anfrage/calculate-price: accepts { produkte: [{selections, resolvedNames, quantity}] }, returns { produkte: [{serverPrice, quantity}], subtotal }
    - POST /api/anfrage/validate-discount: accepts { code, subtotal }, returns DiscountResult
    - Discount validation: ungueltig if code not found, abgelaufen if past gueltig_bis, aufgebraucht if aktuelle_nutzungen >= max_nutzungen, min_bestellwert if subtotal < min, valid if all checks pass
    - Snapshot: produced by calculate-price, includes selections + resolvedNames + serverPrice
  </behavior>
  <action>
    **src/app/api/anfrage/calculate-price/route.ts:** POST Route Handler:
    - Get Payload instance via `getPayload({ config: configPromise })`
    - Parse request body, validate basic structure
    - For each product: call `calculateServerPrice(item.selections, payload)` to get authoritative price
    - Return JSON: `{ produkte: [{ serverPrice, quantity }], subtotal }` where subtotal = sum(serverPrice * quantity)
    - Return 400 on invalid input

    **src/app/api/anfrage/validate-discount/route.ts:** POST Route Handler:
    - Get Payload instance
    - Parse { code, subtotal } from body
    - Query rabattcodes: `payload.find({ collection: 'rabattcodes', where: { code: { equals: code }, aktiv: { equals: true } }, limit: 1 })`
    - Validation chain (order matters):
      1. No result found -> { valid: false, error: 'ungueltig' }
      2. gueltig_bis < now -> { valid: false, error: 'abgelaufen' }
      3. aktuelle_nutzungen >= max_nutzungen -> { valid: false, error: 'aufgebraucht' }
      4. subtotal < min_bestellwert -> { valid: false, error: 'min_bestellwert', minWert }
      5. All passed -> { valid: true, code, typ, wert }
    - Return 400 if code or subtotal missing

    **tests/unit/test-discount-validation.test.ts:** Test the discount validation logic:
    - Extract the validation logic into a pure function `validateDiscountCode(rabatt, subtotal)` in the route file or a helper, so it can be unit tested without HTTP
    - Test: code not found -> ungueltig
    - Test: past gueltig_bis -> abgelaufen
    - Test: max_nutzungen reached -> aufgebraucht
    - Test: subtotal below min_bestellwert -> min_bestellwert with minWert
    - Test: all valid -> returns valid with typ + wert
    - Test: prozent type returns correct wert
    - Test: festbetrag type returns correct wert

    - Verify all required fields present: selections (all KonfiguratorSelections keys), resolvedNames (all display name keys), serverPrice (number > 0)
    - Verify snapshot is JSON-serializable (no functions, no circular refs)
