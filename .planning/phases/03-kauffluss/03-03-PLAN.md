  <action>
    **src/app/api/anfrage/submit/route.ts:** POST Route Handler. The central endpoint that creates the Anfrage:
    1. Get Payload instance via `getPayload({ config: configPromise })`
    2. Parse request body
    3. Validate kontaktdaten with `kontaktSchema.safeParse()` — return 400 with issues on failure
    4. For each product in produkte array:
       - Call `calculateServerPrice(item.selections, payload)` to get authoritative price
       - Build snapshot: { selections: item.selections, resolvedNames: item.resolvedNames, serverPrice }
    5. Calculate gesamtpreis = sum(serverPrice * quantity for each product)
    6. If rabattcode provided:
       - Re-validate discount server-side (query rabattcodes collection, check all conditions)
       - Apply discount: prozent -> gesamtpreis * (1 - wert/100), festbetrag -> gesamtpreis - wert (min 0)
       - Increment aktuelle_nutzungen via `payload.update({ collection: 'rabattcodes', id: rabattId, data: { aktuelle_nutzungen: current + 1 } })`
    7. Generate anfrage_nummer via `generateAnfrageNummer(payload)`
    8. Create Anfrage via `payload.create({ collection: 'anfragen', data: { anfrage_nummer, status: 'neu', produkte (array with konfiguration_snapshot, stueckzahl, einzelpreis), kontaktdaten, gesamtpreis, rabattcode: rabattId or undefined } })`
    9. Return `{ anfrageNummer: anfrage.anfrage_nummer, gesamtpreis }` with status 201
    10. On any error: return 500 with generic error message (do NOT expose internal details)

    **Important:** The kontaktdaten group in Anfragen collection does NOT have a datenschutz field — strip it before saving. The datenschutz checkbox is a form-only consent field.

    - "Wir melden uns innerhalb von 2 Werktagen."

    - Clean up sessionStorage (remove 'christ-kontaktdaten' and any temp data)
