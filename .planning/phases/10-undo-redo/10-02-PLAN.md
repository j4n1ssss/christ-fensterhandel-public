**File 1: `src/components/admin/undo-redo-provider.tsx`**

'use client' component. Creates a React Context holding a `Map<string, UndoRedoStack>` (keyed by docKey = collectionSlug:id).

```typescript
'use client'
import React, { createContext, useContext, useRef, useCallback } from 'react'
import { UndoRedoStack } from '@/components/admin/undo-redo-stack'

interface UndoRedoContextType {
  getStack: (docKey: string) => UndoRedoStack
  resetStack: (docKey: string) => void
}

const UndoRedoContext = createContext<UndoRedoContextType | null>(null)

export function UndoRedoProvider({ children }: { children: React.ReactNode }) {
  const stacksRef = useRef<Map<string, UndoRedoStack>>(new Map())

  const getStack = useCallback((docKey: string) => {
    if (!stacksRef.current.has(docKey)) {
      stacksRef.current.set(docKey, new UndoRedoStack(50))
    }
    return stacksRef.current.get(docKey)!
  }, [])

  const resetStack = useCallback((docKey: string) => {
    stacksRef.current.delete(docKey)
  }, [])

  return (
    <UndoRedoContext.Provider value={{ getStack, resetStack }}>
      {children}
    </UndoRedoContext.Provider>
  )
}

export function useUndoRedoContext() {
  const ctx = useContext(UndoRedoContext)
  if (!ctx) throw new Error('useUndoRedoContext must be used within UndoRedoProvider')
  return ctx
}
```

Hook internals:
1. Get form, [fields, dispatchFields], docInfo, editDepth from Payload hooks
2. Get { getStack } from useUndoRedoContext()
3. Compute docKey via `getDocKey(docConfig?.slug || '', id || '')`
4. Get stack instance via `getStack(docKey)`
5. Use `useRef` for: debounce timer, last snapshot hash (to avoid duplicate pushes), isUndoRedoInProgress flag
6. Use `useState` for: canUndo, canRedo (triggers re-render for button states)

**Snapshot capture (debounced 300ms):**
- Use `useEffect` watching `fields` (from useAllFormFields)
- On change, if `isUndoRedoInProgress.current` is true, skip (prevent capturing the undo/redo itself as a new snapshot)
- Clear existing debounce timer, set new 300ms timeout
- In timeout callback: call `form.getFields()` (NOT the `fields` variable -- avoids stale closure per Pitfall 2), run `createCleanSnapshot()`, compare JSON hash of top-level field values with lastSnapshotHash to avoid duplicate pushes, if different: `stack.push(snapshot)`, update hash ref, call `setCanUndo(stack.canUndo)` and `setCanRedo(stack.canRedo)`

**Save checkpoint:**
- Listen for form submission success. Use `useEffect` to detect when `fields` transitions to a state where all initialValues match values (post-save). Simpler approach: export a `markSaved()` function that the toolbar calls. Actually, detect save by checking if `form.getFields()` has matching value/initialValue after a debounce. Safest: call `stack.markSaved()` when all field.value === field.initialValue (which Payload sets after save).

**Undo action:**
- Set `isUndoRedoInProgress.current = true`
- Get current state for diff: `const before = createCleanSnapshot(form.getFields())`
- Call `stack.undo()` -- if null, return (at floor)
- Dispatch: `dispatchFields({ type: 'REPLACE_STATE', state: snapshot, optimize: false })`
- Call `form.setModified(true)`
- Compute changed fields: compare `before` and `snapshot` top-level value keys
- Highlight changed fields (see below)
- Toast: `toast.info(changedCount === 1 ? '1 Feld rückgängig gemacht' : changedCount + ' Felder rückgängig gemacht', { duration: 2000 })`
- Update canUndo/canRedo state
- After a short delay (50ms), set `isUndoRedoInProgress.current = false`

**Redo action:**
- Same pattern as undo but calls `stack.redo()` and toast says "wiederhergestellt"

**Field highlight:**
- After undo/redo, for each changed field path, query `document.querySelector('[data-path="${path}"]')` or `document.querySelector('[id="field-${path}"]')`
- If element found: add inline style `element.style.transition = 'background-color 500ms ease-out'; element.style.backgroundColor = '#eab30833'` (yellow-500 at 20% opacity)
- After 500ms timeout: `element.style.backgroundColor = 'transparent'`
- After 1000ms timeout: remove inline transition style

**Keyboard shortcuts:**
- Use `useHotkey({ cmdCtrlKey: true, editDepth, keyCodes: ['z'] }, handler)`
- In handler: if `e.shiftKey` call handleRedo, else call handleUndo. Always `e.preventDefault()`.

Return: `{ canUndo, canRedo, handleUndo, handleRedo }`
