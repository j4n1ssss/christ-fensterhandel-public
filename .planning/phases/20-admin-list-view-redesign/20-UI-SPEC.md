# Phase 20 -- UI Design Contract

> Visual and interaction contract for the Admin List View Redesign. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Payload Admin custom.scss (no shadcn in admin) |
| Preset | not applicable (Payload Admin Panel) |
| Component library | No library -- plain React with Payload theme vars |
| Icon library | lucide-react (already used in dashboard-overview.tsx) |
| Font | Payload Admin default (system font stack) |

**Important constraint:** This phase targets the Payload Admin Panel. All styling uses `var(--theme-*)` Payload CSS variables + hex colors from `STATUS_COLORS` via inline styles. No Tailwind, no Shadcn components. CSS classes go in `src/app/(payload)/custom.scss` following BEM-like naming established in Phase 19.

---

## Spacing Scale

Declared values (must be multiples of 4). Inherited from Phase 19 `custom.scss`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Border-left urgency indicator, inline gaps, spec-grid row padding |
| sm | 8px | Tab gaps, badge padding, icon gaps, compact spacing, table cell vertical padding |
| md | 16px | Cell padding (horizontal), section gaps, card padding, table cell horizontal padding |
| lg | 24px | Container padding (`.admin-content`), page-level vertical spacing |
| xl | 32px | Section breaks between filter tabs and table |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Table cell padding: `8px 16px` (grid-aligned values; departs from `dashboard-overview.tsx` `10px 14px` for spec compliance)
- Filter tab padding: `8px 16px` (matches `.tab-panel__trigger` from Phase 19)
- Urgency left-border: `4px solid` (matches `.attention-bar--warn/urgent/critical`)
- 3-dot menu button touch target: `36px` width (matches `.splitbutton__chevron`)

### 4. Attention-Score Bar (`.score-bar`)

| Property | Value |
|----------|-------|
| Container | `width: 60px`, `height: 6px`, `background: var(--theme-elevation-150)`, `border-radius: 3px`, `overflow: hidden` |
| Fill | Inner `div`, `height: 100%`, `border-radius: 3px`, width proportional to `score / maxScoreInView`, colored by score range |
| Formula | `waitingDays * statusWeight` (weights from CONTEXT.md: 3/2/1/0) |
| Tooltip | On hover: `"Score: {N} (Wartezeit: {X} Tage x Gewicht: {W})"` via `title` attribute |

### 5. 3-Dot Menu (`.list-menu`)

| Property | Value |
|----------|-------|
| Trigger | `"..."` button, `36px` wide, centered, stops event propagation (no row click) |
| Dropdown | Absolute positioned below trigger. `min-width: 200px`. Same styling as `.splitbutton__dropdown`. |
| Items | Primary Quick-Action (first from `QUICK_ACTIONS[status]`) with status-color dot. Separator `1px solid var(--theme-elevation-200)`. "Details oeffnen" link. |
| Terminal status | Only "Details oeffnen" (no quick action). |
| COMMENT_REQUIRED | Opens detail view instead of inline action. Menu item label becomes `"{action} (Details oeffnen)"`. |
| After action | Full list reload via re-fetch. No optimistic update. |
| Outside click | Closes dropdown. Escape key closes dropdown. |

### 6. Pagination (`.list-pagination`)

| Property | Value |
|----------|-------|
| Position | Below table, right-aligned |
| Per page | 25 items |
| URL param | `?tab=offen&page=2` |
| Display | `"Seite X von Y"` with Prev/Next buttons |
| Button style | `.btn-ghost` for prev/next. Disabled state when at first/last page. |
| Info text | Left-aligned: `"X Anfragen"` total count, `.text-caption` |

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| shadcn official | none (admin panel does not use shadcn) | not applicable |
| Third-party | none | not applicable |

**Note:** Radix Primitives are NOT used for the 3-dot menu in this phase. The dropdown is implemented as a plain `div` with `position: absolute` + outside-click handler, matching the established `splitbutton__dropdown` pattern from Phase 19. This avoids introducing a new dependency for a simple interaction.

## Accessibility Notes

- Table rows are clickable `<tr>` elements with `role="link"` and `tabIndex={0}` for keyboard navigation
- 3-dot menu trigger is a `<button>` element with `aria-haspopup="true"` and `aria-expanded` state
- Sort headers use `aria-sort="ascending"` / `"descending"` / `"none"`
- Filter tabs use appropriate `role="tablist"` / `role="tab"` / `aria-selected` semantics
- Urgency colors always paired with text labels (never color-only signaling)
- Empty states are announced to screen readers
