**Step 1: Create `src/components/admin/profile-hub-status-cell.tsx`**

Create a new "use client" component that renders a colored badge in the Profile list view:

```typescript
"use client";

import { REQUIRED_HUB_FIELDS } from "@/scripts/validate-hub-fields";

// CellComponentProps type inlined to avoid import issues with Payload internals
interface CellProps {
  cellData?: unknown;
  rowData?: Record<string, unknown>;
}

export function ProfileHubStatusCell({ rowData }: CellProps) {
  const missingFields = REQUIRED_HUB_FIELDS.filter((field) => {
    const value = rowData?.[field];
    return !Array.isArray(value) || value.length === 0;
  });

  const isComplete = missingFields.length === 0;

  return (
    <span
      title={
        isComplete
          ? "Alle Pflicht-Hub-Felder befuellt"
          : `Fehlend: ${missingFields.join(", ")}`
      }
      style={{
        display: "inline-block",
        padding: "2px 8px",
        borderRadius: 4,
        fontSize: 12,
        fontWeight: 600,
        backgroundColor: isComplete ? "#dcfce7" : "#fed7aa",
        color: isComplete ? "#166534" : "#9a3412",
      }}
    >
      {isComplete ? "Vollstaendig" : "Unvollstaendig"}
    </span>
  );
}
```

**Key design decisions (from UI-SPEC.md and CONTEXT.md):**
- Uses `REQUIRED_HUB_FIELDS` from validate-hub-fields.ts (single source of truth, not duplicated)
- Green (#dcfce7/#166534) for complete, orange (#fed7aa/#9a3412) for incomplete
- Native HTML `title` attribute for tooltip (no custom tooltip component)
- 12px font, 600 weight, 4px border-radius, 2px/8px padding
- Display-only, no click handler

**Step 2: Create `src/components/admin/profile-hub-status-filter.tsx`**

Create a filter component that allows admins to filter the Profile list view to show only incomplete profiles. This implements the locked user decision: "Filter-Option in der Profil-Liste: Nur unvollstaendige anzeigen (Dropdown oder Toggle)".

The filter uses a `beforeList` hook approach on the collection, adding a server-side `where` clause that checks all 5 required Hub fields. A profile is "incomplete" if ANY of the 5 fields has `exists: false` or has length 0.

**Implementation approach (choose based on what Payload 3.79 supports best):**

Option A -- If Payload 3 supports `admin.components.beforeListTable` or similar list-level component slots:
Create a "use client" component with a toggle/dropdown that adds URL search params (e.g., `?hub_status=incomplete`), then use a `beforeList` hook in profile.ts to read the param and modify the query:

```typescript
// profile-hub-status-filter.tsx
"use client";
import { useSearchParams, useRouter, usePathname } from "next/navigation";

export function ProfileHubStatusFilter() {
  const searchParams = useSearchParams();
  const router = useRouter();
  const pathname = usePathname();
  const isFiltered = searchParams.get("hub_incomplete") === "true";

  const toggleFilter = () => {
    const params = new URLSearchParams(searchParams.toString());
    if (isFiltered) {
      params.delete("hub_incomplete");
    } else {
      params.set("hub_incomplete", "true");
    }
    router.push(`${pathname}?${params.toString()}`);
  };

  return (
    <button
      onClick={toggleFilter}
      style={{
        padding: "4px 12px",
        borderRadius: 4,
        fontSize: 12,
        fontWeight: 500,
        border: "1px solid var(--theme-elevation-250)",
        backgroundColor: isFiltered ? "#fed7aa" : "transparent",
        color: isFiltered ? "#9a3412" : "var(--theme-elevation-800)",
        cursor: "pointer",
      }}
    >
      {isFiltered ? "Filter aktiv: Nur unvollstaendige" : "Nur unvollstaendige anzeigen"}
    </button>
  );
}
```

Option B -- If list-level component slots are not available, use a `beforeOperation` hook on the collection to intercept `find` operations and check for a custom query param, or implement the filter as a `ui` field with a custom Field component in the list filters area.

**Research during implementation:** Check Payload 3.79 docs/source for `admin.components.BeforeListTable` or `admin.listFilters` to determine the best registration approach. The key requirement is: the filter must be visible in the Profile list view and toggle-able by the admin.
